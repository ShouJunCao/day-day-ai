1.2 Transformer 架构核心思想

Attention 机制 · 位置编码 · 自回归生成
📖 模块一 · 大模型原理入门 · 第 2 节
🎯 核心思想：2017 年，Google 团队发表论文《Attention Is All You Need》，提出了一种完全基于注意力机制的架构——Transformer。它彻底抛弃了传统的循环神经网络（RNN），让所有词可以同时互相关注，实现了真正的并行计算。这个架构后来成为了 GPT、BERT、LLaMA、DeepSeek 等所有大模型的基石，堪称"AI 时代的 CPU"。

一、一篇改变世界的论文

2017 年 6 月 12 日，Google Brain 团队的 8 位研究者（Vaswani、Shazeer、Parmar 等）在 arXiv 上提交了一篇论文，标题大胆而自信："Attention Is All You Need"（你需要的只是注意力）——致敬了经典歌曲 "All You Need Is Love"。

在此之前，处理文本序列（如机器翻译）的主流方法是 RNN 和它的改进版 LSTM。这些网络有一个致命缺陷：必须逐词顺序处理，第 1 个词处理完才能处理第 2 个词。就像排队传话一样，信息从队头传到队尾，越远越失真，而且无法并行加速。

Transformer 的核心主张极为大胆：完全抛弃循环结构，仅用注意力机制处理一切。实验结果令人震惊——翻译质量超越此前最佳模型 2 个 BLEU 分数，训练时间从数周缩短到 3.5 天（8 块 GPU），训练成本降低约 50 倍。

💡 论文作者们的后来

这篇论文的 8 位作者堪称"AI 创业梦之队"：Noam Shazeer 和 Niki Parmar 共同创办了 Character.AI；Aidan Gomez 创办了 Cohere（企业级大模型公司）；Illia Polosukhin 成为 NEAR Protocol 联合创始人。一篇论文，孵化了多家十亿美元级公司。

二、自注意力机制：Transformer 的灵魂

🧠 一个直觉类比

想象你在读这句话：

"这只猫没有过马路，因为它太累了"

当你读到"它"时，大脑会自动回溯判断——"它"指的是"猫"还是"马路"？对人类来说这是本能，但对传统算法很难，因为"猫"和"它"之间隔了好几个词。

自注意力（Self-Attention）就是 Transformer 解决这个问题的方式：让句子中的每个词都能直接"看到"其他所有词，不管距离多远。

🔑 Q、K、V：注意力的三把钥匙

Query（查询） Key（键） Value（值）

每个词被变换成三个向量：

  • Query："我在找什么信息？"——像你在搜索引擎输入的关键词
  • Key："我能提供什么信息？"——像每个网页的标题/标签
  • Value："我的实际内容是什么？"——像网页的实际内容

计算过程：用当前词的 Query 去和所有词的 Key 做匹配（点积），得到相关性分数，经过 Softmax 归一化后，用分数对 Value 加权求和——匹配度高的词贡献更多。

🔧 Java 开发者的直觉

可以把自注意力想象成一种软性 HashMap 查找：Query 是搜索条件，Key 是每个 entry 的键，匹配度高的 value 被加权保留。区别在于 HashMap 是精确匹配（O(1)），而注意力是模糊匹配——每个 key 都或多或少地贡献一些。

三、多头注意力：8 个人同时读文章

单一的注意力只能关注一种关系模式。但语言中的关系是多维的——同一个词可能同时涉及语法关系、指代关系、语义关系等。

多头注意力（Multi-Head Attention）就像让 8 个人同时阅读同一篇文章，每人关注不同方面：一个人关注语法结构，另一个关注指代关系，第三个关注情感色彩……最后把 8 个人的理解合并起来，形成更全面的理解。

8 个注意力头 独立权重矩阵 拼接输出

每个"头"有独立的 Q/K/V 权重矩阵，各自独立计算注意力，得到 8 个输出。然后将 8 个输出拼接起来，再乘以一个权重矩阵压缩回原始维度。

论文中发现：在编码"it"时，一个头关注"the animal"，另一个头关注"tired"——模型通过多个头同时捕获了不同维度的语义关系。

📊 多头注意力：并行捕获不同语义关系

四、位置编码：告诉模型"谁在前谁在后"

Transformer 有一个"先天缺陷"：它不知道词的顺序。对自注意力来说，"猫追狗"和"狗追猫"的输入矩阵完全一样——因为注意力计算只关心"谁和谁相关"，不关心"谁先谁后"。

这就必须人为地给每个位置加上一个"位置信号"——位置编码（Positional Encoding）。

三代位置编码方案

方案原理优点代表模型
正弦/余弦
（原论文）		用不同频率的 sin/cos 函数生成位置向量，像时钟的秒针、分针、时针各自以不同速度转动可外推到未见过的更长序列Transformer 原版
学习型
（Learned PE）		为每个位置学习一个可训练向量，像词嵌入一样从数据中学能从数据中学习位置模式GPT-2/3
RoPE
（旋转位置编码）		用旋转矩阵编码位置，两个向量的点积自然包含相对距离自然编码相对位置，距离越远依赖越弱LLaMA、DeepSeek、Qwen 等现代模型
💡 RoPE 的直觉理解

想象两个钟表的指针：指针转过的角度代表位置。两个指针之间的角度差就是它们的"相对距离"。距离越远，角度差越大，点积（投影）越小——自然地实现了"远距离注意力衰减"。这就是为什么 LLaMA、DeepSeek 等现代模型都选择了 RoPE。

🏆 为什么 Decoder-Only 赢了？

核心原因："预测下一个词"这个简单的训练目标，在模型足够大时，涌现出了理解、推理、代码生成等惊人能力。这就是 Scaling Law 的魔力——一个会写作的人，通过巧妙的提示（prompt），同样可以完成阅读理解；但一个只会阅读理解的人，很难学会写作。

六、自回归生成：一个字一个字地"蹦"

大模型的文本生成方式叫做自回归（Autoregressive）——每次只预测下一个 token，然后把预测结果加入输入，继续预测再下一个。就像手机输入法的"下一个词建议"，但聪明一万倍。

生成过程示例

// 自回归生成：一步一步来 步骤1: 输入 "中国的首都" → 预测 "是" (概率最高) 步骤2: 输入 "中国的首都是" → 预测 "北京" 步骤3: 输入 "中国的首都是北京" → 预测 "。" 步骤4: 输入 "中国的首都是北京。" → 预测 "北京" 步骤5: 输入 "中国的首都是北京。北京" → 预测 "位于" ... 继续直到生成终止符 <EOS>

这就是为什么 GPT 回复看起来是"流式"的——它确实在一个 token 一个 token 地生成。生成 1000 个 token 需要模型运行 1000 次前向传播，这也是推理成本高的根本原因。

八、O(n²) 的代价：为什么上下文窗口不能无限大

自注意力的计算复杂度是 O(n²)——序列长度翻倍，计算量翻四倍，内存也翻四倍。这就是为什么上下文窗口不能无限扩大。

📊 上下文长度 vs 注意力矩阵显存占用（FP16精度，单层单头）
⚡ Flash Attention：让长上下文成为可能

2022 年，Tri Dao 提出 Flash Attention——一种"IO 感知"的注意力算法。核心思想：不把完整的 n×n 注意力矩阵写入显存，而是分块计算，每块在 GPU 高速缓存（SRAM）中完成，大幅减少显存读写次数。

效果：FlashAttention-2 在 A100 上达到了理论最大算力的 50-73%，训练速度比标准注意力快 2-4 倍。FlashAttention-3（2024）在 H100 上更是达到了 82% 的硬件利用率。

总结：一个架构统治 AI 世界

回到 2017 年，没人能预想到一个翻译模型的架构改进，会在此后十年内彻底改变整个人工智能领域。Transformer 的成功可以归结为几个关键因素：

创新点RNN/LSTM 的痛点Transformer 的解法Java 类比
并行计算必须逐词顺序处理所有位置同时计算注意力单线程 for 循环 → 多线程 Stream
长距离依赖信息传 n 步，越远越失真任意两词 1 步直接交互LinkedList 遍历 O(n) → HashMap 直接定位 O(1)
可扩展性训练慢，难以扩大规模天然适配 GPU/TPU 并行单体应用 → 可水平扩展的微服务
多视角理解单一隐藏状态编码一切多头注意力同时捕获多种关系单字段对象 → 多维度数据分析
🎯 Transformer 的本质公式：
Transformer = 自注意力（词间交互）+ 多头注意力（多视角理解）+ 位置编码（顺序感知）+ 前馈网络（非线性变换/知识存储）+ 残差连接（信息保真）+ 层归一化（数值稳定）

这个 2017 年的架构，至今支撑着 GPT、BERT、LLaMA、DeepSeek、Qwen、Claude、ViT（视觉）、AlphaFold2（蛋白质）……一个架构，改变了整个 AI 世界。
