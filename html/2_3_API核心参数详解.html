2.3 API 核心参数详解

temperature · top_p · max_tokens · stop · penalty — 控制模型输出的关键旋钮

💡 核心洞察

大模型的输出不是固定不变的，而是从一个概率分布中"采样"得到的。API 的核心参数就是用来控制这个采样过程的：temperature 决定随机性大小，top_p 决定候选范围，stop sequences 决定何时停止，penalty 参数决定如何避免重复。掌握这些参数，就能精确控制模型的输出风格。

一、参数全景图

Chat Completions API 提供了多个参数来控制模型的输出行为。这些参数可以分为四类：采样控制（temperature、top_p）、输出限制（max_tokens、stop）、重复惩罚（presence_penalty、frequency_penalty）、确定性控制（seed）。理解每个参数的作用，是调优模型输出的基础。

二、temperature — 控制随机性

🌡️ temperature（温度）

范围: 0 ~ 2 默认: 1.0 最常用参数

temperature 是影响模型输出最显著的参数。它控制的是 softmax 函数中概率分布的"锐度"。当 temperature 接近 0 时，模型几乎总是选择概率最高的 token，输出变得高度确定性；当 temperature 增大时，低概率 token 的采样机会增加，输出变得更加多样和"有创意"。

可以这样理解：temperature 就像调音台上的"创造性旋钮"。向左转（低温度），模型变得更严谨、更确定；向右转（高温度），模型变得更发散、更有想象力。但温度太高，输出就会变得混乱甚至无意义。

从数学角度看，temperature 是对 softmax 前的 logits 做缩放：P(token_i) = softmax(logits_i / T)。当 T=0.1 时，高概率 token 的优势被放大到极端，几乎只选最高概率的那个；当 T=2.0 时，所有 token 的概率被"拉平"，连罕见词也有机会被选中。这就是为什么低温度适合需要准确性的任务（代码、数学），而高温度适合需要创造力的任务（故事、诗歌）。

在实际使用中，temperature 的影响因模型而异。有些模型在 temperature=0.5 时就已经相当确定，而另一些模型即使 temperature=1.5 也保持连贯性。建议在新项目开始时，先用几个不同的 temperature 值做对比实验，找到最适合你场景的值。

各场景推荐温度：

三、top_p — 核采样

🎯 top_p（核采样）

范围: 0 ~ 1 默认: 1.0 与 temperature 二选一调

top_p（也叫 nucleus sampling）控制的是采样范围。它不是从所有 token 中采样，而是只从累积概率达到 top_p 的那些 token 中采样。例如 top_p=0.9 表示只从概率总和占前 90% 的 token 中采样，排除了最不常见的那 10%。

与 temperature 的区别在于：temperature 改变的是概率分布的形状（让高概率更高或更平均），top_p 改变的是采样范围的大小（直接排除低概率选项）。两者效果有重叠，通常建议只调其中一个，不要同时调整。

📁 代码位置：workspace/02_api_basics/03_core_params/core_params_explorer.py 
# temperature 和 top_p 的使用示例

from openai import OpenAI

client = OpenAI(
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    api_key=os.getenv("DASHSCOPE_API_KEY")
)

# 示例 1：低温度 → 确定性输出（适合代码生成）
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user", "content": "Python中如何读取CSV文件？"}],
    temperature=0.2,     # 低温度：准确优先
    max_tokens=500,
)

# 示例 2：高温度 → 创造性输出（适合写作）
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user", "content": "写一首关于春天的诗"}],
    temperature=0.9,     # 高温度：创造优先
    top_p=0.95,         # 配合使用
    max_tokens=300,
)

# 示例 3：只用 top_p 控制（推荐方式之一）
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user", "content": "给我一些创业建议"}],
    temperature=1.0,     # 保持默认
    top_p=0.8,          # 只从概率前 80% 的 token 中采样
    max_tokens=500,
)

四、max_tokens 与 stop sequences

📏 max_tokens（最大输出长度）

范围: 1 ~ 模型上限 默认: 模型最大值

max_tokens 限制模型生成的最大 token 数（不包含输入 token）。设置合理的 max_tokens 既能控制成本，又能避免模型生成过多无关内容。需要注意的是，如果模型在达到 max_tokens 之前就自然结束了，finish_reason 会是 "stop"；如果是被截断的，finish_reason 会是 "length"。

🛑 stop sequences（停止序列）

最多 4 个字符串 默认: 无 结构化输出必备

stop sequences 让模型在生成到特定字符串时立即停止。这在结构化输出场景中非常有用，比如让模型生成 JSON 后在 "}" 处停止，或者在多轮对话中用特定分隔符标记每轮回答的边界。

# stop sequences 实战示例

# 场景 1：控制输出格式 — 生成列表后在分隔符处停止
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user",
               "content": "列出5个Python库，每行一个，最后写 END"}],
    stop=["\nEND"],       # 遇到 "\nEND" 就停止
    max_tokens=300,
)

# 场景 2：多轮对话边界控制
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user",
               "content": "回答问题后加上 [DONE]"}],
    stop=["[DONE]"],       # 遇到标记停止
    max_tokens=500,
)

# 场景 3：JSON 输出控制
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user",
               "content": '以JSON格式返回你的名字: {"name": "..."}'}],
    stop=["}\n"],          # JSON 结束后停止
    max_tokens=100,
    temperature=0.1,     # 低温度确保格式正确
)

五、重复惩罚参数

⚖️ presence_penalty 与 frequency_penalty

范围: -2.0 ~ 2.0 默认: 0.0

这两个参数用于控制模型的重复行为，但机制不同。presence_penalty 对已经出现过的 token 施加固定惩罚，不管它出现了几次——它鼓励模型"讨论新话题"。frequency_penalty 则根据 token 出现的频率来惩罚，出现越多惩罚越大——它鼓励模型"少重复"。

图 3：两种重复惩罚机制的区别

# 惩罚参数实战示例

# 场景：长文本生成，避免重复
response = client.chat.completions.create(
    model="qwen-plus",
    messages=[{"role": "user",
               "content": "写一篇关于人工智能的文章，500字"}],
    temperature=0.8,
    presence_penalty=0.5,    # 鼓励讨论新话题
    frequency_penalty=0.3,   # 减少词句重复
    max_tokens=800,
)

print(f"完成原因: {response.choices[0].finish_reason}")
# "stop" = 自然结束, "length" = 达到 max_tokens

六、seed 与确定性输出

seed 参数允许设置随机种子，使得相同输入在相同 seed 下产生相同或极其相似的输出。这对于调试、测试和可复现性非常重要。不过需要注意，seed 并不能保证 100% 确定性——由于底层硬件和软件栈的差异，输出仍可能有微小变化。

seed 在生产环境中的一个重要用途是A/B 测试。当你想比较不同 prompt 的效果时，使用相同的 seed 可以消除随机性带来的干扰，让对比更加公平。另外，在单元测试中，使用固定 seed 可以确保测试结果的可复现性。

# seed 确定性输出示例

# 设置相同的 seed，多次调用得到相似结果
for i in range(3):
    response = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": "Python 之禅"}],
        seed=42,               # 固定随机种子
        temperature=0.7,
        max_tokens=100,
    )
    print(f"Run {i+1}: {response.choices[0].message.content[:60]}")

# 检查 system_fingerprint 确认确定性
# response.system_fingerprint 可用于验证模型版本一致性

六·五、综合实战：参数调优工具

理解了各个参数的作用后，最关键的是能动手实验。以下是一个完整的参数调优工具，可以自动对比不同参数组合下的输出效果，帮助你快速找到最优配置：

📁 代码位置：workspace/02_api_basics/03_core_params/core_params_explorer.py 
"""
API 核心参数实验工具

用法：
    # 对比不同 temperature 的效果
    python3 core_params_explorer.py --param temperature --values 0.0,0.5,1.0,1.5

    # 对比不同 top_p 的效果
    python3 core_params_explorer.py --param top_p --values 0.1,0.5,0.9

    # 演示 stop sequences
    python3 core_params_explorer.py --demo stop_sequences

    # 演示 seed 确定性
    python3 core_params_explorer.py --demo seed_determinism

    # 对比惩罚参数
    python3 core_params_explorer.py --param penalty --presence 0,1 --frequency 0,1
"""

import argparse
import os
from collections import Counter
from openai import OpenAI


def experiment_temperature(client, prompt, temps, runs=3):
    """对比不同 temperature 下模型输出的多样性"""
    for temp in temps:
        print(f"\n📊 temperature = {temp}")
        outputs = []
        for i in range(runs):
            resp = client.chat.completions.create(
                model=os.getenv("MODEL_NAME", "qwen-plus"),
                messages=[{"role": "user", "content": prompt}],
                temperature=temp, max_tokens=100,
            )
            text = resp.choices[0].message.content.strip()
            outputs.append(text)
            print(f"  Run {i+1}: {text[:80]}...")
        unique = len(set(outputs))
        print(f"  多样性: {unique}/{runs} 个不同的输出")


def get_recommended_params(scenario: str) -> dict:
    """根据场景返回推荐参数配置"""
    return {
        "code":    {"temperature": 0.2, "top_p": 0.95, "max_tokens": 2000},
        "chat":    {"temperature": 0.7, "top_p": 0.9,  "max_tokens": 1000},
        "write":   {"temperature": 0.9, "top_p": 0.95, "max_tokens": 1500},
        "json":    {"temperature": 0.1, "top_p": 0.95, "max_tokens": 500},
    }.get(scenario, {"temperature": 0.7, "top_p": 0.9, "max_tokens": 1000})

八、总结

📋 核心要点回顾

1. temperature 控制随机性。值越低输出越确定，值越高输出越多样。代码生成用 0~0.3，创意写作用 0.8~1.2。

2. top_p 控制采样范围。只从累积概率前 top_p 的 token 中采样。与 temperature 二选一调。

3. max_tokens 控制输出长度。合理设置可以控制成本和避免冗余。注意区分 finish_reason 的 "stop" 和 "length"。

4. stop sequences 控制停止时机。最多 4 个停止字符串，结构化输出必备。

5. penalty 参数控制重复。presence_penalty 鼓励新话题，frequency_penalty 减少重复词句。

6. seed 实现确定性。相同 seed + 相同输入 ≈ 相同输出，适合调试和测试。

掌握了 API 核心参数，你就能精确控制模型的输出行为，针对不同场景调优出最佳效果。下一节我们将深入流式输出（SSE）的原理与实现，学习如何在生产环境中高效处理流式响应。
