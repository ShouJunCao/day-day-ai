2.8 实战：封装多模型统一调用 SDK

📋 本节导读

在实际项目中，我们通常需要同时使用多个大模型（如OpenAI、DeepSeek、通义千问等）。每个模型的API格式、参数命名、响应结构都可能不同，这给开发带来了很大的复杂性。本节将实战演练如何封装一个简洁实用的多模型统一调用SDK，让开发者可以用统一的接口调用不同的模型。

一、为什么需要统一SDK？

1.1 多模型调用的痛点

痛点 具体问题 影响
API格式不统一 不同提供商的请求/响应格式不同 需要为每个模型写适配代码
参数命名差异 temperature、top_p等参数名可能不同 容易混淆，难以维护
错误处理复杂 每个API的错误码和错误信息不同 需要分别处理各种错误
配置管理混乱 API Key、Base URL等配置分散 难以统一管理和切换
代码重复 相似的调用逻辑重复编写 维护成本高，容易出错

1.2 统一SDK的价值

💡 核心价值：
  • 简化开发：统一的接口，一套代码调用所有模型
  • 易于切换：只需修改配置，无需改动业务代码
  • 便于对比：同一个问题可以快速在多个模型间对比
  • 集中管理：统一的配置、错误处理、日志记录
  • 提高质量：封装通用逻辑，减少重复代码

二、SDK设计思路

2.1 核心设计原则

原则 说明 实现方式
统一接口 所有模型使用相同的调用方式 定义统一的ChatMessage和ChatResponse
灵活配置 支持多种配置方式 环境变量、配置文件、代码配置
异步设计 支持高并发场景 基于asyncio的异步实现
类型安全 提供完整的类型提示 使用dataclass和类型注解
易于扩展 方便添加新的模型提供商 基于Provider的插件式设计

三、核心代码实现

3.1 数据模型设计

首先定义统一的数据模型，包括请求消息和响应结果：

  • ChatMessage：表示一条聊天消息，包含角色（system/user/assistant）和内容
  • ChatResponse：统一的响应格式，包含生成的文本、使用的模型、Token消耗等信息
  • ModelConfig：模型配置，包括名称、提供商、API密钥、基础URL等
  • Provider：提供商枚举，支持OpenAI、DeepSeek、Qwen等

这些数据结构使用dataclass装饰器，简洁明了，并且支持类型提示，便于IDE自动补全和静态检查。

from dataclasses import dataclass
from typing import Optional, Dict, Any, List
from enum import Enum


class Provider(Enum):
    """模型提供商枚举"""
    OPENAI = "openai"
    DEEPSEEK = "deepseek"
    QWEN = "qwen"
    ZHIPU = "zhipu"


@dataclass
class ChatMessage:
    """聊天消息"""
    role: str  # system, user, assistant
    content: str


@dataclass
class ChatResponse:
    """统一的响应格式"""
    content: str  # 生成的文本
    model: str  # 实际使用的模型名称
    provider: Provider  # 提供商
    usage: Dict[str, int]  # Token使用情况
    raw_response: Any  # 原始响应（可选，用于调试）


@dataclass
class ModelConfig:
    """模型配置"""
    name: str  # 配置名称（用于标识）
    provider: Provider  # 提供商
    model: str  # 模型名称（如gpt-4, deepseek-chat）
    api_key: str  # API密钥
    base_url: Optional[str] = None  # 自定义API端点
    default_params: Optional[Dict[str, Any]] = None  # 默认参数
✨ 设计亮点：使用@dataclass简化数据类定义，提供完整的类型提示。统一的ChatMessage和ChatResponse让所有模型使用相同的数据格式。

3.2 核心客户端实现

实现统一的客户端类，这是SDK的核心：

from typing import Dict, List, Optional
import asyncio


class UnifiedLLMClient:
    """统一的大模型客户端"""
    
    def __init__(self):
        self.models: Dict[str, ModelConfig] = {}
        self.default_model: Optional[str] = None
    
    def register_model(self, config: ModelConfig) -> None:
        """注册模型配置
        
        Args:
            config: 模型配置
        """
        self.models[config.name] = config
        if self.default_model is None:
            self.default_model = config.name
    
    def set_default_model(self, model_name: str) -> None:
        """设置默认模型"""
        if model_name not in self.models:
            raise ValueError(f"模型 {model_name} 未注册")
        self.default_model = model_name
    
    async def chat(
        self,
        messages: List[ChatMessage],
        model: Optional[str] = None,
        temperature: float = 0.7,
        max_tokens: Optional[int] = None,
        **kwargs
    ) -> ChatResponse:
        """统一的聊天接口
        
        Args:
            messages: 消息列表
            model: 模型名称（可选，默认使用default_model）
            temperature: 温度参数
            max_tokens: 最大token数
            **kwargs: 其他参数
        
        Returns:
            ChatResponse: 统一的响应对象
        """
        model_name = model or self.default_model
        if not model_name:
            raise ValueError("未指定模型且未设置默认模型")
        
        if model_name not in self.models:
            raise ValueError(f"模型 {model_name} 未注册")
        
        config = self.models[model_name]
        
        # 根据provider选择不同的调用方式
        if config.provider == ModelProvider.OPENAI:
            return await self._call_openai_compatible(
                config, messages, temperature, max_tokens, **kwargs
            )
        elif config.provider == ModelProvider.DEEPSEEK:
            return await self._call_deepseek(
                config, messages, temperature, max_tokens, **kwargs
            )
        elif config.provider == ModelProvider.QWEN:
            return await self._call_qwen(
                config, messages, temperature, max_tokens, **kwargs
            )
        else:
            raise ValueError(f"不支持的提供商: {config.provider}")

3.3 适配器实现

为每个提供商实现适配器，处理格式转换：

async def _call_openai_compatible(
    self,
    config: ModelConfig,
    messages: List[ChatMessage],
    temperature: float,
    max_tokens: Optional[int],
    **kwargs
) -> ChatResponse:
    """调用OpenAI兼容API"""
    import httpx
    
    base_url = config.base_url or "https://api.openai.com/v1"
    headers = {
        "Authorization": f"Bearer {config.api_key}",
        "Content-Type": "application/json"
    }
    
    # 构建请求体
    payload = {
        "model": config.name,
        "messages": [
            {"role": m.role, "content": m.content} 
            for m in messages
        ],
        "temperature": temperature,
    }
    
    if max_tokens:
        payload["max_tokens"] = max_tokens
    
    payload.update(kwargs)
    
    # 发送异步请求
    async with httpx.AsyncClient(timeout=60.0) as client:
        response = await client.post(
            f"{base_url}/chat/completions",
            headers=headers,
            json=payload
        )
        response.raise_for_status()
        data = response.json()
    
    # 转换为统一响应格式
    return ChatResponse(
        content=data["choices"][0]["message"]["content"],
        model=data["model"],
        provider=config.provider,
        usage=data.get("usage", {}),
        finish_reason=data["choices"][0].get("finish_reason", "stop")
    )
⚠️ 注意：大多数国内模型（DeepSeek、通义千问等）都兼容OpenAI的API格式，因此可以复用_call_openai_compatible方法，只需修改base_url即可。

四、使用示例

4.0 SDK调用流程

下面是SDK的完整调用流程，从用户发起请求到获得响应的整个过程：

调用流程说明：

  1. 用户通过应用层调用chat()方法
  2. SDK检查模型是否已注册，配置是否有效
  3. 根据provider选择对应的适配器
  4. 适配器将统一的ChatMessage转换为具体API的格式
  5. 通过HTTP层发送异步请求
  6. 接收API响应后，适配器将其转换为统一的ChatResponse格式
  7. 返回给用户

async def chat_with_error_handling():
    """带错误处理的聊天"""
    try:
        response = await client.chat(messages)
        return response.content
    except httpx.TimeoutException:
        print("请求超时，请稍后重试")
    except httpx.HTTPStatusError as e:
        if e.response.status_code == 429:
            print("请求过于频繁，请稍后重试")
        elif e.response.status_code == 401:
            print("API Key无效，请检查配置")
        else:
            print(f"HTTP错误: {e.response.status_code}")
    except Exception as e:
        print(f"未知错误: {e}")
    
    return None

5.4 生产环境部署建议

将SDK部署到生产环境时，需要注意以下几个关键点：

  • 密钥管理：永远不要将API Key硬编码在代码中，使用环境变量或密钥管理服务（如AWS Secrets Manager、HashiCorp Vault）
  • 日志记录：记录每次API调用的详细信息，包括模型名称、Token使用量、响应时间等，便于监控和调试
  • 监控告警：设置费用告警阈值，当费用接近预算时及时通知；监控API成功率，发现异常及时处理
  • 限流保护：实现客户端级别的限流，防止单个用户消耗过多资源；设置全局QPS限制，避免触发服务商的限流策略
  • 优雅降级：当API不可用时，提供降级方案，如返回缓存结果、使用更小的模型或返回友好的错误提示
  • 版本管理：为SDK实现版本号，记录变更历史，便于问题追踪和回滚

5.5 成本监控示例

实现一个简单的成本追踪器，帮助监控API调用的费用：

class CostTracker:
    """成本追踪器"""
    
    def __init__(self):
        self.costs = {
            "gpt-4": {"input": 0.03, "output": 0.06},  # 每1K token
            "gpt-3.5-turbo": {"input": 0.0015, "output": 0.002},
            "deepseek-chat": {"input": 0.0007, "output": 0.0014},
        }
        self.total_cost = 0.0
        self.usage_log = []
    
    def record_usage(self, response: ChatResponse):
        """记录一次API调用的成本"""
        model = response.model
        usage = response.usage
        
        if model not in self.costs:
            return
        
        input_cost = (usage.get("prompt_tokens", 0) / 1000) * self.costs[model]["input"]
        output_cost = (usage.get("completion_tokens", 0) / 1000) * self.costs[model]["output"]
        total = input_cost + output_cost
        
        self.total_cost += total
        self.usage_log.append({
            "model": model,
            "input_tokens": usage.get("prompt_tokens", 0),
            "output_tokens": usage.get("completion_tokens", 0),
            "cost": total,
            "timestamp": datetime.now()
        })
    
    def get_summary(self):
        """获取成本汇总"""
        by_model = {}
        for log in self.usage_log:
            model = log["model"]
            if model not in by_model:
                by_model[model] = {"calls": 0, "cost": 0.0}
            by_model[model]["calls"] += 1
            by_model[model]["cost"] += log["cost"]
        
        return {
            "total_cost": self.total_cost,
            "total_calls": len(self.usage_log),
            "by_model": by_model
        }


# 使用示例
tracker = CostTracker()
response = await client.chat(messages)
tracker.record_usage(response)

summary = tracker.get_summary()
print(f"总费用: ${summary['total_cost']:.4f}")
print(f"总调用次数: {summary['total_calls']}")
for model, stats in summary['by_model'].items():
    print(f"  {model}: {stats['calls']}次, ${stats['cost']:.4f}")

def cache_key(messages: List[ChatMessage], model: str) -> str:

class Conversation:
    """对话管理器"""
    
    def __init__(self, client: UnifiedLLMClient, model: Optional[str] = None):
        self.client = client
        self.model = model
        self.history: List[ChatMessage] = []
    
    async def chat(self, user_message: str) -> str:
        """发送消息并获取回复"""
        # 添加用户消息
        self.history.append(ChatMessage(role="user", content=user_message))
        
        # 调用模型
        response = await self.client.chat(self.history, model=self.model)
        
        # 添加助手回复到历史
        self.history.append(ChatMessage(role="assistant", content=response.content))
        
        return response.content
    
    def clear(self):
        """清空对话历史"""
        self.history.clear()

6.2 添加流式输出

async def chat_stream(
    self,
    messages: List[ChatMessage],
    model: Optional[str] = None,
    **kwargs
) -> AsyncGenerator[str, None]:
    """流式聊天接口
    
    Yields:
        str: 每次返回一部分文本
    """
    model_name = model or self.default_model
    config = self.models[model_name]
    
    import httpx
    
    base_url = config.base_url or "https://api.openai.com/v1"
    headers = {
        "Authorization": f"Bearer {config.api_key}",
        "Content-Type": "application/json"
    }
    
    payload = {
        "model": config.name,
        "messages": [
            {"role": m.role, "content": m.content}
            for m in messages
        ],
        "stream": True,  # 启用流式输出
        **kwargs
    }
    
    async with httpx.AsyncClient(timeout=60.0) as client:
        async with client.stream(
            "POST",
            f"{base_url}/chat/completions",
            headers=headers,
            json=payload
        ) as response:
            response.raise_for_status()
            
            # 逐行读取SSE流
            async for line in response.aiter_lines():
                if line.startswith("data: "):
                    data = line[6:]
                    if data == "[DONE]":
                        break
                    
                    import json
                    chunk = json.loads(data)
                    if chunk["choices"][0]["delta"].get("content"):
                        yield chunk["choices"][0]["delta"]["content"]
