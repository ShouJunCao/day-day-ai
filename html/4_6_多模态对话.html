4.6 多模态对话（文本 + 图片 + 语音）

模块四：对话系统设计 预计学习：50 分钟

学习目标

  • 理解多模态对话的核心概念与架构模式
  • 掌握文本、图片、语音三种模态的处理方法
  • 学会设计模态检测器和多模态处理器
  • 了解 ASR（语音识别）、TTS（语音合成）、Vision API 的集成方式
  • 能够构建支持多模态输入输出的对话系统

一、为什么对话系统需要多模态？

在前面的章节中，我们构建了一个完整的任务型对话系统——从意图识别、槽位填充、状态追踪到引导兜底。但这一切都建立在纯文本交互的假设上。现实世界中，用户的表达方式远不止打字。

想想这些场景：

  • 用户拍了一张酒店外观的照片，问："这家怎么样？"
  • 用户发送语音消息："帮我订后天去北京的房"
  • 用户希望系统把预订确认读出来而不是只显示文字
  • 用户上传了一张行程单截图，系统自动提取日期和目的地

多模态对话系统能够理解这些非文本输入，并生成多模态输出。它不是简单的功能叠加，而是交互方式的本质升级——让对话更接近人与人之间的自然交流。

二、三种核心模态

2.1 文本模态

文本是对话系统最基础也最可靠的模态。在前面所有章节中，我们已经建立了完整的文本对话系统——从意图识别到状态追踪。多模态系统中文本模态的角色发生了变化：它不再是唯一的输入输出方式，而是多模态的"通用语言"。

在多模态架构中，所有其他模态最终都会被转换为文本进行处理：图片通过 Vision API 转为描述文本，语音通过 ASR 转为转录文本。然后由对话核心（状态机、意图引擎等）在统一的文本层进行逻辑处理。

2.2 图片模态

图片输入在对话系统中有多种应用场景：

场景输入处理技术输出
视觉问答照片 + "这是什么？"Vision API / VLM文字描述
文档识别行程单截图OCR + NLP 提取结构化数据
商品识别商品照片图像分类 + 搜索商品信息
问题诊断故障截图视觉分析 + 知识库解决方案

2.3 语音模态

语音是交互最自然的方式，尤其是在移动场景中。语音模态涉及两个关键技术：ASR（Automatic Speech Recognition，语音识别）和TTS（Text-to-Speech，语音合成）。

技术方向作用主流方案
ASR语音 → 文本理解用户语音输入Whisper、讯飞、百度
TTS文本 → 语音向用户播报回复Edge TTS、讯飞、OpenAI
⚠️ ASR 的挑战

语音识别的准确率远不如文本输入。方言、噪音、语速、口音都会影响识别结果。因此在设计中必须考虑：识别结果的可信度（置信度分数）和纠错机制（让用户确认或修改识别结果）。

三、模态检测器

多模态对话的第一步是识别用户输入的模态类型。在最简单的情况下，系统可以通过 MIME 类型判断（image/jpeg 是图片，audio/mpeg 是语音）。但更常见的情况是系统只收到文本，需要从文本内容中推断用户是否期望多模态交互。

04_dialogue_system/06_multimodal_dialogue/multimodal_dialogue.py
"""
multimodal_dialogue.py — 多模态对话处理器
支持文本、图片、语音三种模态的对话输入和输出。
"""
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional
from datetime import datetime


class Modality(Enum):
    """输入模态类型"""
    TEXT = "text"
    IMAGE = "image"
    VOICE = "voice"


@dataclass
class MultimodalInput:
    """多模态输入"""
    modality: Modality
    text: str = ""
    image_url: Optional[str] = None
    audio_url: Optional[str] = None
    transcribed_text: Optional[str] = None  # 语音转文字结果

    @property
    def primary_content(self) -> str:
        """获取主要文本内容"""
        if self.text:
            return self.text
        if self.transcribed_text:
            return self.transcribed_text
        return ""


@dataclass
class ModalityDetector:
    """
    模态检测器：识别用户输入的模态类型。
    支持从 MIME 类型或文本关键词推断。
    """

    def detect_from_text(self, text: str) -> Modality:
        """从文本内容推断意图模态"""
        image_keywords = ["图片", "照片", "截图", "看一下", "看看"]
        voice_keywords = ["语音", "说一下", "读给我听"]

        for kw in image_keywords:
            if kw in text:
                return Modality.IMAGE
        for kw in voice_keywords:
            if kw in text:
                return Modality.VOICE

        return Modality.TEXT

    def detect_from_content_type(self, content_type: str) -> Modality:
        """从 MIME 类型推断模态"""
        if "image" in content_type:
            return Modality.IMAGE
        if "audio" in content_type:
            return Modality.VOICE
        return Modality.TEXT

四、多模态处理器

模态检测之后，系统需要根据输入模态选择合适的处理策略。多模态处理器的核心职责是：接收多模态输入 → 转换为统一表示 → 调用合适的处理链 → 生成多模态输出。

    def process(
        self,
        user_input: MultimodalInput,
        context: Optional[dict] = None,
    ) -> MultimodalResponse:
        """处理多模态输入，生成回复"""
        context = context or {}

        # 根据输入模态选择处理策略
        if user_input.modality == Modality.IMAGE:
            return self._handle_image(user_input, context)
        elif user_input.modality == Modality.VOICE:
            return self._handle_voice(user_input, context)
        else:
            return self._handle_text(user_input, context)

    def _handle_image(
        self, input: MultimodalInput, context: dict
    ) -> MultimodalResponse:
        """处理图片输入"""
        if self.enable_vision and input.image_url:
            # 调用视觉模型分析图片
            analysis = f"[图片分析] 已收到图片: {input.image_url}"
            return MultimodalResponse(
                text=f"我看到了您发送的图片。\n{input.text}\n{analysis}",
            )
        return MultimodalResponse(
            text="抱歉，当前不支持图片分析功能。",
        )

    def _handle_voice(
        self, input: MultimodalInput, context: dict
    ) -> MultimodalResponse:
        """处理语音输入"""
        if self.enable_asr and input.audio_url:
            transcribed = input.transcribed_text or "[语音识别结果]"
            return MultimodalResponse(
                text=f"您说的是：{transcribed}\n我正在为您处理...",
            )
        return MultimodalResponse(
            text="抱歉，当前不支持语音输入功能。",
        )

处理器的关键在于优雅降级。当某个模态的功能未启用或不可用时，系统不应该崩溃，而应该返回友好的提示信息。这也是为什么我们在构造函数中用 enable_vision、enable_voice、enable_asr 三个开关来控制功能可用性。

五、与对话核心的集成

多模态处理器不是替代对话核心（状态机、意图引擎），而是包裹在核心之外，负责输入输出的模态转换。

下面展示了完整的模态检测 + 多模态处理流程：

"""
多模态对话 — 完整使用示例
"""
from multimodal_dialogue import (
    MultimodalInput, Modality, ModalityDetector,
    MultimodalDialogueProcessor,
)


def demo_multimodal_processing():
    # 启用所有功能
    processor = MultimodalDialogueProcessor(
        enable_vision=True,
        enable_voice=True,
        enable_asr=True,
    )

    # 场景 1：纯文本
    inp1 = MultimodalInput(
        modality=Modality.TEXT,
        text="今天天气怎么样？"
    )
    resp1 = processor.process(inp1)
    print(f"输入: {inp1.text}")
    print(f"回复: {resp1.text}")

    # 场景 2：图片 + 文本
    inp2 = MultimodalInput(
        modality=Modality.IMAGE,
        text="这张图片里有什么？",
        image_url="https://example.com/photo.jpg",
    )
    resp2 = processor.process(inp2)
    print(f"输入: [图片] {inp2.text}")
    print(f"回复: {resp2.text}")

    # 场景 3：语音输入
    inp3 = MultimodalInput(
        modality=Modality.VOICE,
        audio_url="https://example.com/voice.mp3",
        transcribed_text="我想预订后天去北京的酒店",
    )
    resp3 = processor.process(inp3)
    print(f"输入: [语音] → \"{inp3.transcribed_text}\"")
    print(f"回复: {resp3.text}")

七、生产环境集成指南

7.1 Vision API 集成

在生产环境中，图片理解需要接入 Vision API。OpenAI 的 GPT-4o、Claude 的 Vision 能力、以及开源的 Qwen-VL 都是常见选择。

import httpx
import base64
from pathlib import Path
from typing import Optional


class VisionClient:
    """
    Vision API 客户端：将图片发送给多模态 LLM，
    获取图片描述或分析结果。
    """

    def __init__(
        self,
        api_key: str,
        base_url: str = "https://api.openai.com/v1",
        model: str = "gpt-4o",
    ):
        self.api_key = api_key
        self.base_url = base_url
        self.model = model

    def analyze_image(
        self,
        image_path: str,
        prompt: str = "请详细描述这张图片的内容",
    ) -> str:
        """分析图片并返回文本描述"""
        # 读取图片并编码为 base64
        image_data = Path(image_path).read_bytes()
        b64 = base64.b64encode(image_data).decode()

        payload = {
            "model": self.model,
            "messages": [
                {"role": "user", "content": [
                    {"type": "text", "text": prompt},
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:image/jpeg;base64,{b64}",
                        },
                    },
                ]},
            ],
            "max_tokens": 500,
        }

        with httpx.Client(timeout=30) as client:
            resp = client.post(
                f"{self.base_url}/chat/completions",
                headers={"Authorization": f"Bearer {self.api_key}"},
                json=payload,
            )
            resp.raise_for_status()
            return resp.json()["choices"][0]["message"]["content"]

7.2 ASR 集成

语音识别推荐使用 Whisper API 或本地部署的 Whisper 模型。OpenAI 的 Whisper API 支持多种语言和音频格式，集成简单。

import httpx
from pathlib import Path


class ASRClient:
    """
    Whisper ASR 客户端：将音频文件转为文本。
    支持多种音频格式：mp3、wav、ogg、flac 等。
    """

    def __init__(
        self,
        api_key: str,
        base_url: str = "https://api.openai.com/v1",
        model: str = "whisper-1",
        language: str = "zh",
    ):
        self.api_key = api_key
        self.base_url = base_url
        self.model = model
        self.language = language

    def transcribe(self, audio_path: str) -> dict:
        """
        将音频文件转为文本。

        Returns:
            {"text": "...", "language": "...", "duration": float}
        """
        path = Path(audio_path)
        if not path.exists():
            raise FileNotFoundError(f"音频文件不存在: {audio_path}")

        with open(path, "rb") as f:
            files = {
                "file": (path.name, f, "audio/mpeg"),
            }
            data = {
                "model": self.model,
                "language": self.language,
                "response_format": "verbose_json",
            }

            with httpx.Client(timeout=60) as client:
                resp = client.post(
                    f"{self.base_url}/audio/transcriptions",
                    headers={"Authorization": f"Bearer {self.api_key}"},
                    data=data,
                    files=files,
                )
                resp.raise_for_status()
                return resp.json()

7.3 TTS 集成

语音合成可以使用 Edge TTS（免费）或 OpenAI TTS API。Edge TTS 支持多种语言和声音，无需 API Key。

class TTSClient:
    """
    语音合成客户端：将文本转为语音。
    支持 Edge TTS 和 OpenAI TTS 两种方式。
    """

    def __init__(self, provider: str = "edge"):
        """
        provider: "edge" (免费) 或 "openai" (付费)
        """
        self.provider = provider

    async def synthesize_edge(
        self,
        text: str,
        voice: str = "zh-CN-XiaoxiaoNeural",
        output_path: str = "output.mp3",
    ) -> str:
        """使用 Edge TTS 生成语音"""
        import edge_tts
        comm = edge_tts.Communicate(text, voice)
        await comm.save(output_path)
        return output_path

八、对比：纯文本 vs 多模态对话

维度纯文本系统多模态系统
用户输入方式只能打字打字、拍照、语音
交互自然度中等（需要用户组织文字）高（说话或拍照即可）
实现复杂度低（纯 API 调用）高（需要集成多种服务）
成本低（仅 LLM Token）高（ASR + Vision + TTS）
延迟1-3 秒2-8 秒（取决于模态）
适用场景标准客服、后台管理移动端、老年用户、现场服务
💡 渐进式多模态策略

不必一步到位。建议分阶段实施：阶段一（纯文本）→ 阶段二（+ 语音输入 ASR）→ 阶段三（+ 语音输出 TTS）→ 阶段四（+ 图片理解 Vision）。每增加一种模态，都需要评估 ROI（用户活跃度提升 vs 成本增加）。

九、常见误区

误区一：多模态等于必须全部支持

很多团队觉得"多模态"就意味着必须同时支持文字、图片、语音。其实应该根据用户群体和场景选择性地启用。比如后台管理系统只需要文本，老年用户应用重点做语音，旅游应用重点做图片。

误区二：忽略模态转换的损耗

ASR 的识别准确率不是 100%，Vision API 的描述也不是完美的。每次模态转换都会引入信息损失。因此系统需要在关键节点让用户确认或修改转换结果，而不是直接使用可能错误的识别结果。

误区三：不考虑离线场景

ASR 和 Vision API 都需要网络连接。在弱网或离线环境下，系统需要有降级方案——比如允许用户手动输入文本替代语音识别，或者提示"请在网络良好的环境下使用图片功能"。

十、小结

多模态对话是对话系统从"能用"到"好用"的关键一步。通过理解图片、语音等非文本输入，系统能够覆盖更广泛的用户群体和使用场景。多模态处理器的核心设计原则是：模态检测 → 统一文本层处理 → 多模态输出，这样既保持了对话核心的一致性，又提供了丰富的交互体验。
