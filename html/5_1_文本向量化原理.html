5.1 文本向量化原理（Word → Sentence → Contextual Embedding）

模块五：向量数据库与 Embedding 预计学习：60 分钟

学习目标

  • 理解文本向量化的本质：将文字转换为机器可计算的数值向量
  • 掌握词向量演进的三个阶段：One-Hot → TF-IDF → 密集词向量
  • 学会从词向量组合成句向量的方法：词袋平均、TF-IDF 加权、注意力加权
  • 理解上下文感知嵌入（Contextual Embedding）的原理和优势
  • 掌握 Transformer Encoder 层的核心组件：自注意力、残差连接、层归一化、前馈网络

一、为什么需要文本向量化？

计算机只能理解数字，不理解文字。要让计算机处理文本——搜索、分类、比较相似度——我们必须先把文字变成数字。这就是文本向量化的核心目的。Embedding（嵌入）就是这个过程的学术名称。

文本向量化有三个层次，对应不同粒度的语义表示：

层次单位示例代表方法
词级（Word）单个词"猫"、"银行"One-Hot, TF-IDF, Word2Vec
句级（Sentence）完整句子"我喜欢猫"词袋平均, Sentence-BERT
上下文（Contextual）词 + 上下文"银行"在"河边"vs"取钱"中不同BERT, GPT, ELMo

二、词级向量化：从稀疏到密集

2.1 One-Hot 编码（独热编码）

最原始的向量化方法。每个词对应一个维度，该位置为 1，其余为 0。简单但问题严重：维度灾难（10 万词汇表 = 10 万维向量）和无语义信息（"猫"和"狗"的向量距离与"猫"和"桌子"完全一样）。

❌ One-Hot 的问题

词汇表 100,000 词 → 向量 100,000 维

99.999% 的位置都是 0（极度稀疏）

"猫"和"狗"的余弦相似度 = 0（无法表达相似性）

✅ Word2Vec 的改进

统一压缩到 300 维（密集向量）

每个维度都有非零值

"猫"和"狗"的余弦相似度 ≈ 0.85（语义相近）

2.2 Word2Vec：从稀疏到密集的跨越

Word2Vec 是 Google 在 2013 年提出的经典算法，它将词映射到一个低维密集向量空间中。在这个空间里，语义相似的词向量距离更近——"猫"和"狗"的向量比"猫"和"桌子"更接近。这就是向量化真正意义的起点。

Word2Vec 有两种训练模式：CBOW（根据上下文预测中心词）和Skip-gram（根据中心词预测上下文）。Skip-gram 对低频词的效果更好，因此在实际项目中更常用。训练完成后，每个词都对应一个 300 维左右的密集向量，向量之间的余弦相似度反映了词语之间的语义相似度。

2.3 TF-IDF（词频-逆文档频率）

"""
word_embedding.py — 词向量化实现
包含 One-Hot 编码和 TF-IDF 的完整实现。
"""
import math
from collections import Counter
from typing import Optional


class OneHotEncoder:
    """
    独热编码器：将词语转换为独热向量。

    原理：
    - 首先从语料中构建词汇表（每个词分配一个唯一的索引）
    - 对于每个词，创建一个长度为词汇表大小的向量
    - 该词对应的索引位置设为 1，其余位置为 0
    """

    def __init__(self):
        """初始化：词汇表字典和词汇表大小"""
        self.vocab: dict[str, int] = {}
        self.vocab_size: int = 0

    def build_vocab(self, corpus: list[list[str]]) -> None:
        """
        从语料库中构建词汇表。

        参数：
        - corpus (list[list[str]]): 语料库，每个元素是一个分词后的句子
          示例：[["我", "喜欢", "猫"], ["我", "喜欢", "狗"]]
        """
        idx = 0  # 索引计数器：从 0 开始
        for sentence in corpus:
            for word in sentence:
                if word not in self.vocab:
                    self.vocab[word] = idx  # 词 → 索引的映射
                    idx += 1
        self.vocab_size = idx  # 更新词汇表大小

    def encode(self, word: str) -> Optional[list[int]]:
        """
        将单个词编码为独热向量。

        返回：
        - list[int] 或 None: 独热向量，未知词返回 None
        """
        if word not in self.vocab:
            return None  # 未知词（OOV），无法编码

        word_idx = self.vocab[word]  # 获取词对应的索引
        one_hot_vector = [0] * self.vocab_size  # 创建全 0 向量
        one_hot_vector[word_idx] = 1  # 将词对应的位置设为 1
        return one_hot_vector

2.2 TF-IDF（词频-逆文档频率）

TF-IDF 是 One-Hot 的重要改进。它不再只关心"词是否出现"，而是衡量"这个词对当前文档有多重要"。核心思想：在一个文档中频繁出现但在其他文档中很少出现的词，对该文档具有高区分度。

指标公式含义
TF（词频） TF = 词在文档中出现次数 / 文档总词数 衡量词在当前文档中的重要性
IDF（逆文档频率） IDF = log(总文档数 / 包含该词的文档数) + 1 衡量词的全局稀有程度
TF-IDF TF-IDF = TF × IDF 综合局部重要性和全局稀有性

举例说明：假设我们有 100 篇文档。"的"出现在 99 篇中（IDF 很低），"量子计算"出现在 2 篇中（IDF 很高）。即使在某篇文档中"的"出现了 50 次（高 TF），其 TF-IDF 仍然很低——因为它对这篇文档没有区分度。相反，"量子计算"即使只出现 3 次，TF-IDF 也会很高——因为它强烈指示这篇文档的主题。

class TFIDFEncoder:
    """
    TF-IDF 编码器：计算词频-逆文档频率权重。

    公式：
    - TF(t, d) = 词 t 在文档 d 中出现的次数 / 文档 d 的总词数
    - IDF(t) = log(总文档数 / 包含词 t 的文档数) + 1
    - TF-IDF(t, d) = TF(t, d) × IDF(t)
    """

    def __init__(self):
        """初始化：词汇表、IDF 字典、文档数"""
        self.vocab: dict[str, int] = {}
        self.vocab_size: int = 0
        self.idf: dict[str, float] = {}
        self.num_docs: int = 0

    def build_vocab(self, corpus: list[list[str]]) -> None:
        """从语料库构建词汇表并计算 IDF 值"""
        self.num_docs = len(corpus)  # 文档总数

        # 第一步：构建词汇表
        idx = 0
        for doc in corpus:
            for word in doc:
                if word not in self.vocab:
                    self.vocab[word] = idx
                    idx += 1
        self.vocab_size = idx

        # 第二步：计算 IDF
        doc_freq: dict[str, int] = {}  # 词的文档频率
        for doc in corpus:
            unique_words = set(doc)  # 一个词在一个文档中只计一次
            for word in unique_words:
                doc_freq[word] = doc_freq.get(word, 0) + 1

        # IDF 公式：log(N / df) + 1
        for word, df in doc_freq.items():
            self.idf[word] = math.log(self.num_docs / df) + 1

    def transform(self, doc: list[str]) -> list[float]:
        """
        将文档转换为 TF-IDF 向量。

        计算步骤：
        1. 计算每个词的 TF（词频）
        2. 乘以对应的 IDF 值
        3. 按词汇表索引填充向量
        """
        tfidf_vector = [0.0] * self.vocab_size  # 全 0 向量
        doc_length = len(doc)  # 文档总词数
        if doc_length == 0:
            return tfidf_vector

        term_freq = Counter(doc)  # 统计词频

        for word, count in term_freq.items():
            if word not in self.vocab:
                continue

            tf = count / doc_length  # 词频
            idf = self.idf.get(word, 1.0)  # 逆文档频率
            tfidf_value = tf * idf  # TF-IDF

            word_idx = self.vocab[word]
            tfidf_vector[word_idx] = tfidf_value  # 填充向量

        return tfidf_vector

三、句级向量化：从词到句子

词向量只能表示单个词的含义。但大多数应用需要理解整个句子的语义——比如语义搜索、文本聚类、相似度计算、问答匹配等。如何将一组词向量组合成一个句子向量？这是句级向量化的核心问题。

3.1 词袋平均（Bag-of-Words Average）

最简单的方法：将所有词的向量取算术平均。这种方法实现简单、计算快速，但有两个明显缺陷：忽略词序（"猫吃鱼"和"鱼吃猫"的向量完全相同）和等权对待所有词（停用词"的"和关键词"猫"的权重一样）。尽管如此，在很多简单场景下词袋平均的效果出乎意料地好，常被用作基线方法。

3.2 TF-IDF 加权平均

改进版：用 TF-IDF 值作为权重对词向量加权平均。重要词（高 TF-IDF）对句向量的贡献更大，停用词（低 TF-IDF）的贡献更小。这种方法在不引入额外模型复杂度的情况下，显著提升了句向量的质量。

3.3 注意力加权（Attention-Weighted）

更高级的方法：引入注意力机制，让模型自动学习哪些词更重要。核心思想是定义一个查询向量（Query），计算每个词与查询的点积作为注意力分数，再用 softmax 归一化为权重。注意力加权相比固定权重（如 TF-IDF）的优势在于，它能够根据具体查询动态调整各词的重要性——同一句话在不同查询下会有不同的加权结果。

"""
sentence_embedding.py — 句向量化实现
包含词袋平均、TF-IDF 加权、注意力加权三种方法。
"""
import math
from collections import Counter
from typing import Optional


class BagOfWordsAverage:
    """
    词袋平均编码器：通过对词向量取平均生成句向量。

    原理：
    sentence_vector = mean([word_vector(w) for w in sentence])
    """

    def __init__(self, word_vectors):
        """参数：word_vectors 提供词向量的对象"""
        self.word_vectors = word_vectors

    def encode(self, sentence: list[str]) -> Optional[list[float]]:
        """将句子编码为句向量（词向量平均）"""
        valid_vectors: list[list[float]] = []

        for word in sentence:
            vec = self.word_vectors.get_vector(word)
            if vec is not None:
                valid_vectors.append(vec)

        if not valid_vectors:
            return None  # 没有有效词

        dim = len(valid_vectors[0])
        avg_vector = [0.0] * dim

        # 对每个维度求和
        for vec in valid_vectors:
            for i in range(dim):
                avg_vector[i] += vec[i]

        # 求平均：总和 / 有效词数
        num_words = len(valid_vectors)
        avg_vector = [v / num_words for v in avg_vector]

        return avg_vector


class SimpleAttentionSentence:
    """
    简单注意力加权编码器：模拟注意力机制生成句向量。

    原理（简化版注意力）：
    1. 定义一个查询向量 query（代表"句子主题"）
    2. 计算每个词向量与 query 的点积作为注意力分数
    3. 用 softmax 将分数归一化为注意力权重
    4. 用注意力权重对词向量加权求和

    公式：
    attention_score(w) = dot(query, word_vector(w))
    attention_weight(w) = softmax(attention_scores)
    sentence_vector = sum(attention_weight(w) * word_vector(w))
    """

    def __init__(self, word_vectors, dim: int = 8):
        """参数：word_vectors 提供词向量的对象，dim 向量维度"""
        self.word_vectors = word_vectors
        # 查询向量：初始化为均匀向量
        self.query = [1.0 / math.sqrt(dim)] * dim

    def _softmax(self, scores: list[float]) -> list[float]:
        """
        计算 softmax，将分数归一化为概率分布。

        公式：softmax(x_i) = exp(x_i) / sum(exp(x_j))
        注意：使用减去最大值的技巧防止数值溢出
        """
        max_score = max(scores)
        exp_scores = [math.exp(s - max_score) for s in scores]
        total = sum(exp_scores)
        return [e / total for e in exp_scores]

    def _dot_product(self, vec1: list[float], vec2: list[float]) -> float:
        """计算两个向量的点积：dot(a, b) = sum(a_i * b_i)"""
        result = 0.0
        for i in range(len(vec1)):
            result += vec1[i] * vec2[i]
        return result

    def encode(self, sentence: list[str]) -> Optional[list[float]]:
        """将句子编码为注意力加权句向量"""
        valid_vectors: list[list[float]] = []
        for word in sentence:
            vec = self.word_vectors.get_vector(word)
            if vec is not None:
                valid_vectors.append(vec)

        if not valid_vectors:
            return None

        dim = len(valid_vectors[0])

        # 步骤 1：计算注意力分数
        scores = []
        for vec in valid_vectors:
            score = self._dot_product(self.query, vec)  # query · word_vector
            scores.append(score)

        # 步骤 2：softmax 归一化为注意力权重
        weights = self._softmax(scores)

        # 步骤 3：加权求和
        result_vector = [0.0] * dim
        for vec, weight in zip(valid_vectors, weights):
            for i in range(dim):
                result_vector[i] += weight * vec[i]

        return result_vector

四、上下文感知嵌入：解决一词多义

前面所有方法的共同缺陷是：同一个词在任何上下文中都有相同的向量。但"银行"在"人民银行"中指金融机构，在"银行门口"中指河岸——这就是一词多义（Polysemy）问题。传统词向量（Word2Vec、GloVe）为每个词分配固定向量，天然无法处理一词多义。

上下文感知嵌入的核心思想：词的向量不是固定的，而是根据其左右邻居动态生成的。同一个词在不同上下文中会有不同的向量表示。2018 年 ELMo 和 BERT 的发布标志着这一范式的转折点——从此，词的表示不再独立于上下文，而是上下文的一部分。

4.1 双向上下文编码

简化版的 BiLSTM 思想：从左到右扫描积累前向上下文，从右到左扫描积累后向上下文，最终向量 = 词向量 + 前向上下文 + 后向上下文。这种方法虽然不如 BERT 的 Transformer 架构复杂，但已经能够部分解决一词多义问题——同一个词在不同位置的向量会因为累积的上下文不同而有所差异。

4.2 自注意力层（Self-Attention）

Transformer 的核心组件。让序列中的每个词都能"看到"所有其他词的信息，并动态决定关注哪些词。相比双向上下文编码的固定权重累积，自注意力的优势在于自适应权重——模型会根据当前词和每个邻居的相关性自动分配关注度，而不是平等地累积所有上下文。

05_vector_embedding/01_word_embedding/contextual_embedding.py
"""
contextual_embedding.py — 上下文感知嵌入实现
包含双向上下文编码、自注意力层、Transformer Encoder 层。
"""
import math
from typing import Optional


class BiContextualEncoder:
    """
    双向上下文编码器：根据词的左右邻居生成动态词向量。

    原理（简化版 BiLSTM）：
    - 前向传递：从左到右扫描，每个词的向量累加左侧上下文信息
    - 后向传递：从右到左扫描，每个词的向量累加右侧上下文信息
    - 最终向量 = 词向量 + 前向上下文 + 后向上下文
    """

    def __init__(self, word_vectors, context_weight: float = 0.3):
        """
        参数：
        - word_vectors: 提供基础词向量的对象
        - context_weight: 上下文信息的影响权重 (0-1)
        """
        self.word_vectors = word_vectors
        self.context_weight = context_weight

    def encode_sentence(self, sentence: list[str]) -> list[Optional[list[float]]]:
        """将整个句子编码为上下文感知的词向量列表"""
        # 获取所有基础词向量
        base_vectors = [self.word_vectors.get_vector(w) for w in sentence]
        dim = next((len(v) for v in base_vectors if v), 0)
        if dim == 0:
            return base_vectors

        # 前向传递：从左到右累积上下文
        forward_context = [[0.0] * dim for _ in range(len(sentence))]
        accumulated = [0.0] * dim
        for i in range(len(sentence)):
            if base_vectors[i] is not None:
                for j in range(dim):
                    accumulated[j] += base_vectors[i][j]
                for j in range(dim):
                    forward_context[i][j] = accumulated[j]

        # 后向传递：从右到左累积上下文
        backward_context = [[0.0] * dim for _ in range(len(sentence))]
        accumulated = [0.0] * dim
        for i in range(len(sentence) - 1, -1, -1):
            if base_vectors[i] is not None:
                for j in range(dim):
                    accumulated[j] += base_vectors[i][j]
                for j in range(dim):
                    backward_context[i][j] = accumulated[j]

        # 合并：词向量 + 前向上下文 + 后向上下文
        contextual_vectors = []
        for i in range(len(sentence)):
            if base_vectors[i] is None:
                contextual_vectors.append(None)
                continue
            bv = base_vectors[i]
            assert bv is not None
            result = [0.0] * dim
            for j in range(dim):
                result[j] = (
                    bv[j]
                    + self.context_weight * forward_context[i][j]
                    + self.context_weight * backward_context[i][j]
                )
            # 归一化
            norm = math.sqrt(sum(v * v for v in result))
            if norm > 0:
                result = [v / norm for v in result]
            contextual_vectors.append(result)

        return contextual_vectors

4.2 自注意力层（Self-Attention）

Transformer 的核心组件。让序列中的每个词都能"看到"所有其他词的信息，并动态决定关注哪些词。

组件含义类比
Query (Q)我想知道什么你去图书馆"搜索"一本书
Key (K)我有什么信息每本书的"书名和标签"
Value (V)我的实际内容书的"正文内容"

注意力计算公式：Attention(Q, K, V) = softmax(QK^T / √d_k) · V

直觉：用 Query 去"搜索"所有 Key，找到最匹配的 Key 对应的 Value，加权求和得到上下文感知的表示。

class SelfAttentionLayer:
    """
    自注意力层：计算序列中所有词对之间的注意力权重。

    公式：Attention(Q, K, V) = softmax(QK^T / √d_k) · V

    对于每个词：
    1. 用 Q 向量去"查询"所有词的 K 向量
    2. 点积越大，表示相关性越高
    3. 用 softmax 归一化为注意力权重
    4. 用权重对所有词的 V 向量加权求和
    """

    def __init__(self, dim: int = 8):
        """参数：dim 向量维度"""
        self.dim = dim
        # Q/K/V 权重（简化为对角矩阵）
        self.W_q = [1.0] * dim
        self.W_k = [1.0] * dim
        self.W_v = [1.0] * dim

    def forward(self, input_vectors: list[list[float]]) -> list[list[float]]:
        """
        前向传播：计算自注意力输出。

        计算步骤（对每个位置 i）：
        1. 计算 query_i = W_q · input_i
        2. 对每个位置 j，计算 key_j = W_k · input_j
        3. 计算注意力分数 score_ij = dot(query_i, key_j) / √dim
        4. 对分数做 softmax 得到注意力权重
        5. 输出 = sum(weight_ij * value_j)
        """
        n = len(input_vectors)
        if n == 0:
            return []

        dim = len(input_vectors[0])

        # 步骤 1：计算所有位置的 Q, K, V
        Q = [self._apply_weight(v, self.W_q) for v in input_vectors]
        K = [self._apply_weight(v, self.W_k) for v in input_vectors]
        V = [self._apply_weight(v, self.W_v) for v in input_vectors]

        # 缩放因子：防止点积过大导致 softmax 梯度消失
        scale = math.sqrt(dim)

        output = []
        for i in range(n):
            # 计算位置 i 对所有位置 j 的注意力分数
            scores = [self._dot_product(Q[i], K[j]) / scale for j in range(n)]
            weights = self._softmax(scores)  # softmax 归一化

            # 加权求和 Value
            result_vec = [0.0] * dim
            for j in range(n):
                for d in range(dim):
                    result_vec[d] += weights[j] * V[j][d]
            output.append(result_vec)

        return output

4.3 Transformer Encoder 层

完整的 Encoder 层将自注意力与前馈网络组合，并加入残差连接和层归一化。这是 BERT、GPT 等模型的基本构建块。

class TransformerEncoderLayer:
    """
    Transformer Encoder 层：自注意力 + 前馈网络的组合。

    结构：
    Input → 自注意力 → 残差连接 → 层归一化 → 前馈网络 → 残差连接 → 层归一化 → Output
    """

    def __init__(self, dim: int = 8):
        """参数：dim 向量维度"""
        self.dim = dim
        self.attention = SelfAttentionLayer(dim)

    def _layer_norm(self, vectors: list[list[float]]) -> list[list[float]]:
        """
        层归一化：对每个位置的向量做归一化。

        公式：
        mean = sum(x_i) / n
        variance = sum((x_i - mean)^2) / n
        normalized = (x_i - mean) / √(variance + ε)
        """
        epsilon = 1e-5
        normalized = []
        for vec in vectors:
            mean = sum(vec) / len(vec)
            variance = sum((x - mean) ** 2 for x in vec) / len(vec)
            norm_vec = [(x - mean) / math.sqrt(variance + epsilon) for x in vec]
            normalized.append(norm_vec)
        return normalized

    def _residual_add(self, original, transformed):
        """残差连接：output = original + transformed"""
        return [[o + t for o, t in zip(orig, trans)]
                for orig, trans in zip(original, transformed)]

    def _feed_forward(self, vectors, hidden_dim=16):
        """
        前馈网络：两层 MLP + ReLU。
        结构：x → Linear(dim→hidden) → ReLU → Linear(hidden→dim)
        """
        output = []
        for vec in vectors:
            # 第一层 + ReLU
            hidden = []
            for h in range(hidden_dim):
                val = sum(vec[d] * math.sin((h+1)*(d+1)*0.1) for d in range(self.dim))
                hidden.append(max(0.0, val))  # ReLU
            # 第二层
            result = [sum(hidden[h] * math.sin((h+1)*(d+1)*0.1)
                         for h in range(hidden_dim)) for d in range(self.dim)]
            output.append(result)
        return output

    def forward(self, input_vectors: list[list[float]]) -> list[list[float]]:
        """
        Encoder 层的前向传播。

        完整流程：
        Input → 自注意力 → 残差 → LayerNorm → 前馈网络 → 残差 → LayerNorm → Output
        """
        # Sub-layer 1: 自注意力 + 残差 + 层归一化
        attn_output = self.attention.forward(input_vectors)
        residual1 = self._residual_add(input_vectors, attn_output)
        norm1 = self._layer_norm(residual1)

        # Sub-layer 2: 前馈网络 + 残差 + 层归一化
        ffn_output = self._feed_forward(norm1)
        residual2 = self._residual_add(norm1, ffn_output)
        norm2 = self._layer_norm(residual2)

        return norm2

五、向量化方法对比

经过五个阶段的演进，文本向量化从最原始的独热编码发展到了上下文感知的 Transformer 架构。下面是各方法的全面对比：

六、小结

文本向量化经历了从稀疏到密集、从静态到动态、从词级到上下文的演进。每种方法都解决前一种方法的痛点：

  1. One-Hot 解决了"文字→数字"的基本问题，但维度灾难和无语义信息使其不实用
  2. TF-IDF 通过加权突出了重要词，但仍然是稀疏表示
  3. Word2Vec 将词映射到密集向量空间，语义相似的词向量接近，但无法处理一词多义
  4. 上下文感知（BERT 等） 让词向量随上下文动态变化，彻底解决了一词多义问题
