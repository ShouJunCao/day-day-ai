5.2 主流 Embedding 模型对比（OpenAI、BGE-M3、GTE、M3E、Jina）

学习目标

  • 学会根据语言、成本、维度、开源性选择合适模型

一、为什么需要对比 Embedding 模型？

在上一节 5.1 中，我们学习了文本向量化的原理和演进。但原理归原理，实际开发中面临一个更现实的问题：到底该选哪个模型？

当前 Embedding 模型生态百花齐放——OpenAI 的 API 模型、智源 BAAI 的开源模型、阿里的 GTE、MokaAI 的 M3E、Jina AI 的多语言模型……每个模型都有各自的优劣和适用场景。选错了模型，可能导致效果差、成本高、延迟大、语言不支持等一系列问题。

本节将系统对比 6 款主流 Embedding 模型，从维度、成本、语言支持、MTEB 评分等多个维度进行横向比较，帮助你做出最佳选择。

📌 MTEB 是什么？

MTEB（Massive Text Embedding Benchmark）是目前最权威的 Embedding 模型评估基准，涵盖 8 大任务类型（分类、聚类、检索、STS 等）、58 个数据集。分数越高表示综合语义理解能力越强。注意：MTEB 评分主要用于横向对比，不代表你的具体业务场景的绝对效果。

三、各模型详解

3.1 OpenAI Embedding

OpenAI 的 Embedding 模型是API 调用最简单的选择。一行代码就能完成编码，无需下载模型、无需 GPU、无需任何基础设施。但代价是：闭源、付费、不支持中文优化。

text-embedding-3-small 是性价比最高的选择，1536 维，MTEB 评分 64.6，每百万 Token 仅 0.02 美元。对于英文项目且预算有限的团队来说，这是最务实的选择。text-embedding-3-large 则以 3072 维和 67.3 的 MTEB 评分成为精度最高的商用模型，但价格是 small 版本的 6.5 倍，适合对精度要求极高的场景。

05_vector_embedding/02_embedding_models/embedding_models.py
"""
embedding_models.py — 主流 Embedding 模型统一客户端
提供 OpenAI、BGE-M3、GTE、M3E、Jina 五种模型的统一调用接口。
"""
import os
import math
from typing import Optional
from dataclasses import dataclass, field


@dataclass
class EmbeddingResult:
    """
    Embedding 结果：封装向量、模型名、维度等元数据。

    属性：
    - vector (list[float]): 嵌入向量
    - model (str): 使用的模型名称
    - dimension (int): 向量维度
    - usage (dict): Token 使用量（API 模型）
    """
    vector: list[float]
    model: str
    dimension: int
    usage: dict = field(default_factory=dict)

    def cosine_similarity(self, other: "EmbeddingResult") -> float:
        """
        计算与另一个 EmbeddingResult 的余弦相似度。

        公式：cos(a, b) = (a · b) / (|a| × |b|)

        返回：
        - float: 余弦相似度，范围 [-1, 1]
        """
        dot_product = sum(a * b for a, b in zip(self.vector, other.vector))
        norm_a = math.sqrt(sum(a * a for a in self.vector))
        norm_b = math.sqrt(sum(b * b for b in other.vector))
        if norm_a == 0 or norm_b == 0:
            return 0.0
        return dot_product / (norm_a * norm_b)


class OpenAIEmbedding:
    """
    OpenAI Embedding 客户端：调用 OpenAI API 获取文本嵌入。

    支持的模型：
    - text-embedding-3-small：1536 维，成本低，适合大多数场景
    - text-embedding-3-large：3072 维，精度高，适合需要细粒度区分的场景
    """

    AVAILABLE_MODELS = {
        "text-embedding-3-small": 1536,
        "text-embedding-3-large": 3072,
        "text-embedding-ada-002": 1536,
    }

    def __init__(self, model: Optional[str] = None):
        """从环境变量读取配置，模型名称可选"""
        self.api_key = os.getenv("OPENAI_API_KEY", "")
        self.base_url = os.getenv("OPENAI_BASE_URL", "https://api.openai.com/v1")
        self.model = model or os.getenv("OPENAI_EMBED_MODEL", "text-embedding-3-small")
        self.dimension = self.AVAILABLE_MODELS.get(self.model, 1536)

    def embed(self, text: str) -> EmbeddingResult:
        """
        将单个文本转换为嵌入向量。

        返回：
        - EmbeddingResult: 包含向量、模型名、维度等信息
        """
        import httpx

        with httpx.Client(timeout=30) as client:
            resp = client.post(
                f"{self.base_url}/embeddings",
                headers={
                    "Authorization": f"Bearer {self.api_key}",
                    "Content-Type": "application/json",
                },
                json={"model": self.model, "input": text},
            )
            resp.raise_for_status()
            data = resp.json()

            vector = data["data"][0]["embedding"]
            usage = data.get("usage", {})

            return EmbeddingResult(
                vector=vector,
                model=self.model,
                dimension=self.dimension,
                usage=usage,
            )

3.2 BGE-M3（智源）

BGE-M3 是目前综合能力最强的开源 Embedding 模型。由北京智源人工智能研究院（BAAI）发布，支持 100+ 种语言、8192 Token 长文本，同时返回稠密向量、稀疏向量和多向量三种表示。

特性说明
多语言支持 100+ 种语言，中文效果优秀
多粒度词级、句级、段落级都能处理
多任务稠密（Dense）、稀疏（Sparse）、多向量（Multi-vector）
最大长度8192 Token，适合长文档
部署方式HuggingFace Transformers 本地推理，需 GPU
 
class BGEM3Embedding:
    """
    BGE-M3 Embedding 客户端：使用 HuggingFace Transformers 本地推理。

    特点：
    - 支持 100+ 种语言
    - 同时返回稠密、稀疏、多向量三种表示
    - 最大序列长度 8192
    """

    DIMENSION = 1024

    def __init__(self, model_name: Optional[str] = None):
        """参数：model_name 模型路径，默认 BAAI/bge-m3"""
        self.model_name = model_name or "BAAI/bge-m3"
        self._tokenizer = None
        self._model = None

    def _load_model(self):
        """延迟加载模型和分词器"""
        from transformers import AutoTokenizer, AutoModel
        self._tokenizer = AutoTokenizer.from_pretrained(self.model_name)
        self._model = AutoModel.from_pretrained(self.model_name)
        self._model.eval()  # 评估模式（禁用 Dropout）

    def embed(self, text: str) -> EmbeddingResult:
        """将文本编码为 1024 维稠密向量"""
        import torch

        if self._model is None:
            self._load_model()

        inputs = self._tokenizer(
            text, padding=True, truncation=True,
            max_length=8192, return_tensors="pt",
        )

        with torch.no_grad():
            outputs = self._model(**inputs)
            # 使用 [CLS] token 的向量作为句子表示
            sentence_embeddings = outputs.last_hidden_state[:, 0]
            # L2 归一化
            sentence_embeddings = torch.nn.functional.normalize(
                sentence_embeddings, p=2, dim=1
            )

        vector = sentence_embeddings[0].tolist()

        return EmbeddingResult(
            vector=vector,
            model=self.model_name,
            dimension=self.DIMENSION,
        )

3.3 GTE（阿里通义）

GTE（General Text Embedding）是阿里巴巴通义实验室的通用文本嵌入模型。在 MTEB 英文基准上排名第二（66.8），仅次于 OpenAI 的 large 模型，且完全开源免费。GTE 系列包含 base（768 维）和 large（1024 维）两个版本，其中 large 版本在检索和聚类任务上表现尤为出色。

GTE 的优势在于英文效果极好，但中文支持相对 BGE-M3 和 m3e-base 稍弱。如果你的项目以英文为主，同时需要一定程度的中文能力，GTE 是一个性价比极高的选择。它基于 HuggingFace Transformers 部署，只需要一台带 GPU 的服务器即可运行。

3.4 M3E（MokaAI）

M3E 是国内社区 MokaAI 开源的多语言嵌入模型。虽然在 MTEB 上的绝对分数不是最高（63.2），但在中文语义理解任务上表现特别出色，且模型体积小（base 仅 ~400MB），适合资源受限的场景。

M3E 最大的优势是中文优化——它使用了大量中文语料进行训练，在中文文本的语义匹配、分类、检索等任务上，效果甚至超过了 MTEB 分数更高的英文模型。如果你的项目是纯中文场景，且没有 GPU 资源（只能用 CPU 推理），M3E 的 base 版本是最务实的选择。它在 CPU 上的推理速度也明显优于 BGE-M3 和 GTE。

3.5 Jina Embeddings

Jina AI 的 Embedding 模型特点是超长文本支持（8192 Token）和task-specific 模式——同一个模型可以根据任务类型（检索查询、检索文档、文本匹配、文本分类）返回不同的向量表示。

task-specific 模式是 Jina 的独特优势。以检索场景为例，查询文本（短、意图明确）和文档文本（长、内容丰富）的嵌入策略应该不同。Jina v3 允许你在调用时指定 task 参数，模型会内部调整注意力权重，使查询和文档在同一个向量空间中更容易匹配。这种设计避免了传统方案中需要为查询和文档分别训练不同模型的问题。

五、向量相似度计算

Embedding 的核心价值是将语义相似度转化为向量距离。我们通常使用余弦相似度来衡量两个向量的语义接近程度。

度量方法公式范围特点
余弦相似度cos = (a·b) / (|a|×|b|)[-1, 1]最常用，不受向量长度影响
欧氏距离d = √(Σ(aᵢ-bᵢ)²)[0, ∞)值越小越相似，受向量长度影响
点积dot = Σ(aᵢ×bᵢ)(-∞, ∞)快速但不归一化，适合已归一化的向量

"""
model_comparison.py — Embedding 模型对比工具
不依赖实际 API 调用，使用模拟数据演示模型间的差异。
"""
import math
from dataclasses import dataclass, field


@dataclass
class ModelSpec:
    """
    Embedding 模型规格：封装模型的关键参数。

    属性：
    - name (str): 模型名称
    - provider (str): 提供方
    - dimension (int): 向量维度
    - max_tokens (int): 最大输入 Token 数
    - languages (list[str]): 支持的语言
    - pricing_per_1m (float): 每百万 Token 价格（美元）
    - is_open_source (bool): 是否开源
    - hf_model_id (str): HuggingFace 模型 ID
    - mteb_score (float): MTEB 基准评分（0-100）
    """
    name: str
    provider: str
    dimension: int
    max_tokens: int
    languages: list[str] = field(default_factory=list)
    pricing_per_1m: float = 0.0
    is_open_source: bool = False
    hf_model_id: str = ""
    mteb_score: float = 0.0

    def estimate_cost(self, num_texts: int, avg_tokens: float) -> float:
        """
        估算处理指定文本量的成本。

        参数：
        - num_texts (int): 文本数量
        - avg_tokens (float): 平均每条文本的 Token 数

        返回：
        - float: 估算成本（美元）
        """
        total_tokens = num_texts * avg_tokens
        return (total_tokens / 1_000_000) * self.pricing_per_1m


class SimilarityCalculator:
    """
    向量相似度计算器：提供多种向量距离度量方法。
    """

    @staticmethod
    def cosine_similarity(vec1: list[float], vec2: list[float]) -> float:
        """
        计算余弦相似度。

        公式：cos(a, b) = (a · b) / (|a| × |b|)

        返回：
        - float: 范围 [-1, 1]
            1.0 完全相同，-1.0 完全相反，0 正交
        """
        dot = sum(a * b for a, b in zip(vec1, vec2))
        norm1 = math.sqrt(sum(a * a for a in vec1))
        norm2 = math.sqrt(sum(b * b for b in vec2))
        if norm1 == 0 or norm2 == 0:
            return 0.0
        return dot / (norm1 * norm2)

    @staticmethod
    def euclidean_distance(vec1: list[float], vec2: list[float]) -> float:
        """
        计算欧氏距离。
        公式：dist = √(Σ(aᵢ - bᵢ)²)
        值越小表示越相似。
        """
        return math.sqrt(sum((a - b) ** 2 for a, b in zip(vec1, vec2)))

    @staticmethod
    def dot_product(vec1: list[float], vec2: list[float]) -> float:
        """
        计算点积（内积）。
        公式：dot = Σ(aᵢ × bᵢ)
        """
        return sum(a * b for a, b in zip(vec1, vec2))

六、API 模型 vs 本地模型

选择 Embedding 模型时，一个重要的决策点是：用 API 还是本地部署？这不仅是技术选择，更是成本、隐私、可控性的综合权衡。

维度API 模型（OpenAI/Jina）本地模型（BGE-M3/GTE/M3E）
部署难度极低（一行代码）中高（需要 GPU 和 Transformers）
成本按量计费，量大贵前期 GPU 投入，量大便宜
延迟网络延迟 + API 处理仅推理延迟（GPU 上通常更快）
数据隐私数据发送到第三方完全本地，数据不出境
可定制性无法微调可以微调适配特定领域
可扩展性API 有速率限制完全可控

经验法则：如果你的月调用量在 1000 万 Token 以下，API 模型的总成本通常低于自建（考虑 GPU 硬件、运维人力、电费等隐性成本）。超过这个阈值后，本地部署的经济优势开始显现。但经济成本不是唯一因素——数据隐私和合规要求在金融、医疗等行业中可能是决定性因素。

⚠️ 数据隐私警告

如果你的数据包含敏感信息（用户隐私、商业机密等），强烈建议使用本地开源模型。API 模型会将你的文本发送到第三方服务器，存在数据泄露和合规风险。对于金融、医疗、政务等敏感行业，本地模型是唯一合规的选择。

七、常见误区

误区一：维度越高越好

高维度确实能捕获更多语义信息，但也意味着更高的存储成本和更长的计算时间。实际上，768 维和 1024 维的效果差异在大多数业务场景中并不明显。建议先用低维模型验证效果，确认需要更高精度时再升级。

误区二：MTEB 分数越高一定越好

MTEB 是综合基准，但不等于你的具体业务场景的效果。如果你的业务是中文客服搜索，MTEB 分数较低的 m3e-base 可能比分数更高的 GTE 效果更好。始终用你自己的业务数据做评估。

误区三：一个模型适用所有场景

检索查询（短文本）和检索文档（长文本）往往需要不同的嵌入策略。Jina v3 的 task-specific 模式就是一个很好的尝试——同一个模型，不同任务返回不同向量。在生产环境中，为不同任务选择不同模型或不同模式往往能得到更好的效果。

八、小结

Embedding 模型的选择需要综合考虑语言支持、维度、成本、开源性、MTEB 分数五个维度。对于中文项目，BGE-M3 是综合能力最强的开源选择；对于英文项目且预算充足，OpenAI large 是精度最高的选择；对于资源受限的中文短文本场景，m3e-base 是性价比最高的选择。
