模块五：向量数据库与 Embedding 预计学习：50 分钟

学习目标

  • 掌握六款主流向量数据库的核心特点和适用场景

  • 能够估算向量存储的磁盘开销

一、为什么需要专门的向量数据库？

传统关系数据库擅长精确匹配和范围查询，但面对"找最相似的 N 个向量"时完全力不从心。100 万个 768 维向量中逐个计算距离意味着上百万次浮点运算，秒级到分钟级的延迟对在线服务完全不可接受。

向量数据库通过预建索引（HNSW、IVF 等）将检索复杂度从 O(N) 降到 O(log N)，压缩到毫秒级。这就是它区别于传统数据库的根本所在。

能力MySQL向量数据库
精确匹配✅ 擅长（B-Tree 索引）✅ 也支持（主键索引）
范围查询✅ 擅长❌ 不擅长
相似度搜索❌ 全表扫描，O(N)✅ 近似搜索，O(log N)
高维向量❌ 不支持✅ 原生支持
元数据过滤✅ WHERE 条件✅ Payload 过滤
📌 核心理解

向量数据库不是用来替代传统数据库的，而是互补的。实际生产中，向量数据库负责语义相似度搜索，传统数据库负责事务处理和精确查询，两者通过业务 ID 关联。这种架构既发挥了各自的长处，又避免了把不适合的工作强加给某一方。

二、六款主流向量数据库对比

当前向量数据库生态百花齐放，从 Meta 的本地搜索库 FAISS，到零配置的嵌入式数据库 Chroma，再到企业级分布式方案 Milvus，以及全托管云服务 Pinecone，选择非常多。下面系统对比六款主流方案。

三、各数据库详解与代码示例

3.1 FAISS — 最快的本地向量搜索库

FAISS（Facebook AI Similarity Search）是 Meta 开源的向量搜索库。它不是传统意义上的数据库，而是一个 C++ 库，通过 Python 接口调用。它的核心优势是速度——在单机上就能处理十亿级向量的检索，延迟可以控制在毫秒级。

但 FAISS 的局限性也很明显：没有内置持久化（需要手动保存索引文件到磁盘）、不支持分布式（单机上限）、不支持元数据过滤（只能返回向量 ID，需要外部映射到业务数据）。因此 FAISS 最适合的场景是：单机部署、对延迟极度敏感、向量规模在千万级以内、不需要复杂过滤的检索服务。

维度说明
优势速度最快、内存效率高、支持多种索引类型（Flat/IVF/HNSW/PQ）
劣势无持久化、无分布式、无元数据过滤、需要外部管理 ID 映射
适用单机、对延迟敏感、中小规模（< 1000 万向量）
05_vector_embedding/03_vector_databases/vector_databases.py
"""
vector_databases.py — 主流向量数据库统一客户端
提供 FAISS、Milvus、Qdrant、Chroma、Pinecone、pgvector
六种向量数据库的统一操作接口。
"""
import os
import json
import math
from typing import Optional
from dataclasses import dataclass, field


@dataclass
class SearchResult:
    """
    向量搜索结果：封装匹配文档及其相似度分数。

    属性：
    - id (str): 文档 ID
    - score (float): 相似度分数（越高越相似）
    - metadata (dict): 文档元数据
    """
    id: str
    score: float
    metadata: dict = field(default_factory=dict)


class FAISSClient:
    """
    FAISS 客户端：基于 Meta FAISS 库的本地向量搜索。

    特点：
    - 速度最快（C++ 底层，高度优化）
    - 不支持内置持久化（需要手动保存索引文件）
    - 无分布式能力（单机使用）
    - 支持多种索引类型（Flat、IVF、HNSW、PQ）
    """

    def __init__(self, dimension: int, index_type: str = "flat"):
        """
        初始化 FAISS 客户端。

        参数：
        - dimension (int): 向量维度
        - index_type (str): 索引类型，可选 "flat"、"ivf"、"hnsw"
        """
        self.dimension = dimension
        self.index_type = index_type
        self.index = None  # FAISS 索引对象（延迟初始化）
        self.id_mapping: dict[int, str] = {}  # 内部 ID → 业务 ID
        self.metadata_store: dict[str, dict] = {}
        self._next_id = 0

    def _init_index(self):
        """延迟初始化 FAISS 索引"""
        import faiss
        if self.index_type == "flat":
            # Flat 索引：暴力搜索，最精确
            self.index = faiss.IndexFlatL2(self.dimension)
        elif self.index_type == "ivf":
            # IVF 索引：倒排文件，适合大数据量
            q = faiss.IndexFlatL2(self.dimension)
            self.index = faiss.IndexIVFFlat(q, self.dimension, 100)
        elif self.index_type == "hnsw":
            # HNSW 索引：分层可导航小世界图
            self.index = faiss.IndexHNSWFlat(self.dimension, 16)

    def insert(self, vectors: list[list[float]], ids: list[str],
               metadata: Optional[list[dict]] = None) -> int:
        """
        批量插入向量。

        参数：
        - vectors: 向量列表
        - ids: 对应的业务 ID 列表
        - metadata: 元数据列表（可选）
        """
        import numpy as np
        if self.index is None:
            self._init_index()
        np_vectors = np.array(vectors, dtype=np.float32)
        # IVF 索引需要先训练
        if self.index_type == "ivf" and not self.index.is_trained:
            self.index.train(np_vectors)
        for i in range(len(vectors)):
            self.id_mapping[self._next_id] = ids[i]
            if metadata and i < len(metadata):
                self.metadata_store[ids[i]] = metadata[i]
            self._next_id += 1
        self.index.add(np_vectors)
        return len(vectors)

    def search(self, query_vector: list[float], top_k: int = 5) -> list[SearchResult]:
        """
        搜索最相似的 K 个向量。

        参数：
        - query_vector: 查询向量
        - top_k: 返回结果数
        """
        import numpy as np
        if self.index is None:
            return []
        np_query = np.array([query_vector], dtype=np.float32)
        distances, indices = self.index.search(np_query, top_k)
        results = []
        for i, idx in enumerate(indices[0]):
            if idx == -1:
                continue  # -1 表示没有更多结果
            bid = self.id_mapping.get(int(idx), str(idx))
            meta = self.metadata_store.get(bid, {})
            sim = 1.0 / (1.0 + distances[0][i])
            results.append(SearchResult(id=bid, score=sim, metadata=meta))
        return results

3.2 Chroma — 最易用的嵌入式向量数据库

Chroma 的设计理念是"零配置、即用即走"。pip install 后直接在代码中使用，还支持内置的 sentence-transformers 嵌入生成。内置持久化——指定目录即可自动保存，下次启动自动加载。

Chroma 非常适合本地开发、快速原型和小型项目。但当数据量超过百万级时，性能和可扩展性成为瓶颈。

"""Chroma 客户端：零配置嵌入式向量数据库"""
import chromadb


class ChromaClient:
    """
    Chroma 客户端：轻量级嵌入式向量数据库。

    特点：
    - 无需外部服务（嵌入式运行）
    - Python 原生 API
    - 支持持久化到磁盘
    - 内置文本嵌入生成
    """

    def __init__(self, collection_name: str = "default",
                 persist_dir: str = "./chroma_data"):
        self.collection_name = collection_name
        self.persist_dir = persist_dir
        self.client = None
        self.collection = None

    def _init_client(self):
        """延迟初始化：创建持久化客户端 + 集合"""
        self.client = chromadb.PersistentClient(path=self.persist_dir)
        self.collection = self.client.get_or_create_collection(
            name=self.collection_name,
            metadata={"hnsw:space": "cosine"},  # 余弦相似度
        )

    def insert(self, vectors, ids, metadata=None) -> int:
        """批量插入"""
        if self.client is None:
            self._init_client()
        self.collection.add(ids=ids, embeddings=vectors, metadatas=metadata)
        return len(ids)

    def search(self, query_vector, top_k=5):
        """向量搜索"""
        if self.client is None:
            return []
        results = self.collection.query(
            query_embeddings=[query_vector], n_results=top_k,
        )
        return [SearchResult(
            id=results["ids"][0][i],
            score=1.0 / (1.0 + results["distances"][0][i]),
            metadata=results["metadatas"][0][i]
        ) for i in range(len(results["ids"][0]))]

3.3 Qdrant — 高性能 Rust 向量数据库

Qdrant 用 Rust 编写，性能接近 FAISS 但功能更完整——支持持久化、元数据过滤、分布式部署。它的 payload filtering 功能特别强大，可以在向量搜索的同时按元数据条件筛选，比如"只搜索 tech 分类下的相似文档"。这对 RAG 系统非常有用——你可以先过滤掉不相关的时间范围或权限区域，再在子集中做向量搜索。

Qdrant 的部署比 Chroma 复杂一些（需要启动独立服务进程），但比 Milvus 简单得多（不需要 Kubernetes）。它提供了 REST API 和 gRPC 两种接口，Python SDK 用起来也很方便。

"""Qdrant 客户端：Rust 高性能向量数据库"""
from qdrant_client import QdrantClient as QdrantSDK
from qdrant_client.models import (Distance, VectorParams,
                                   PointStruct, Filter, FieldCondition)


class QdrantClientWrapper:
    """Qdrant 客户端封装：支持 payload 过滤"""

    def __init__(self, collection_name="default",
                 url="http://localhost:6333"):
        self.collection_name = collection_name
        self.url = url
        self.client = None

    def _init_client(self, dimension=768):
        """延迟初始化：创建集合（余弦相似度）"""
        self.client = QdrantSDK(url=self.url)
        exists = any(c.name == self.collection_name
                     for c in self.client.get_collections().collections)
        if not exists:
            self.client.create_collection(
                collection_name=self.collection_name,
                vectors_config=VectorParams(
                    size=dimension, distance=Distance.COSINE),
            )

    def insert(self, vectors, ids, metadata=None) -> int:
        """批量插入：Point(id, vector, payload) + upsert"""
        points = [
            PointStruct(
                id=ids[i], vector=vectors[i],
                payload=metadata[i] if metadata and i < len(metadata) else {}
            ) for i in range(len(vectors))
        ]
        self.client.upsert(
            collection_name=self.collection_name, points=points)
        return len(points)

    def search(self, query_vector, top_k=5, filter_dict=None):
        """搜索 + payload 过滤"""
        if self.client is None:
            self._init_client()
        qf = None
        if filter_dict:
            qf = Filter(must=[
                FieldCondition(key=k, match={"value": v})
                for k, v in filter_dict.items()
            ])
        results = self.client.search(
            collection_name=self.collection_name,
            query_vector=query_vector, limit=top_k, query_filter=qf,
        )
        return [SearchResult(id=str(h.id), score=h.score,
                metadata=h.payload or {}) for h in results]

3.4 Milvus — 企业级分布式向量数据库

Milvus 是目前功能最全面的开源向量数据库，支持分布式部署、GPU 加速、多种索引类型、多租户隔离。这些特性使它成为大规模向量检索的首选。

但代价是部署和运维复杂度最高——需要 Kubernetes 环境、需要配置多个组件。如果团队没有 K8s 运维经验，建议先从 Qdrant 或 Chroma 开始，业务量增长后再迁移。

"""Milvus 客户端：开源分布式向量数据库"""
from pymilvus import MilvusClient as MilvusSDK


class MilvusClientWrapper:
    """Milvus：分布式、高可用、GPU 加速"""

    def __init__(self, collection_name="default",
                 uri="http://localhost:19530"):
        self.collection_name = collection_name
        self.uri = uri
        self.client = None

    def _init_client(self, dimension=768):
        self.client = MilvusSDK(uri=self.uri)
        if not self.client.has_collection(self.collection_name):
            self.client.create_collection(
                collection_name=self.collection_name,
                dimension=dimension, metric_type="COSINE")

    def insert(self, vectors, ids, metadata=None) -> int:
        """批量插入"""
        if self.client is None:
            self._init_client()
        data = []
        for i in range(len(vectors)):
            item = {"id": ids[i], "vector": vectors[i]}
            if metadata and i < len(metadata):
                item["meta"] = metadata[i]
            data.append(item)
        self.client.insert(
            collection_name=self.collection_name, data=data)
        return len(data)

    def search(self, query_vector, top_k=5):
        """向量搜索"""
        if self.client is None:
            return []
        results = self.client.search(
            collection_name=self.collection_name,
            data=[query_vector], anns_field="vector",
            limit=top_k, output_fields=["id", "meta"],
        )
        return [SearchResult(id=str(h["id"]),
                score=h["distance"],
                metadata=h.get("meta", {})) for h in results[0]]

3.5 Pinecone — 全托管云服务

Pinecone 的核心卖点是"不用管基础设施"。无需部署、无需运维、自动扩缩容，按量付费。对于资源有限的团队，这是最省心的选择。

代价是闭源、供应商锁定、长期成本高于自建。数据量长期稳定且规模不大时，自建方案（Chroma 或 Qdrant）更经济。

"""Pinecone 客户端：全托管云端向量数据库"""
import pinecone


class PineconeClientWrapper:
    """Pinecone：全托管、自动扩展、Serverless"""

    def __init__(self, index_name="default", api_key=None):
        self.index_name = index_name
        self.api_key = api_key or os.getenv("PINECONE_API_KEY", "")
        self.index = None

    def _init_client(self):
        pinecone.init(api_key=self.api_key, environment="us-east1-aws")
        self.index = pinecone.Index(self.index_name)

    def upsert(self, vectors, ids, metadata=None) -> int:
        """批量插入/更新"""
        if self.index is None:
            self._init_client()
        items = []
        for i in range(len(vectors)):
            item = {"id": ids[i], "values": vectors[i]}
            if metadata and i < len(metadata):
                item["metadata"] = metadata[i]
            items.append(item)
        self.index.upsert(vectors=items)
        return len(items)

    def query(self, query_vector, top_k=5):
        """向量搜索"""
        if self.index is None:
            return []
        results = self.index.query(
            vector=query_vector, top_k=top_k, include_metadata=True,
        )
        return [SearchResult(id=m["id"], score=m["score"],
                metadata=m.get("metadata", {}))
                for m in results["matches"]]

3.6 pgvector — PostgreSQL 的向量扩展

如果你的项目已经在用 PostgreSQL，pgvector 是最自然的选择。只需要一条 CREATE EXTENSION vector 命令，你的数据库就能存储和检索向量了。更重要的是，你可以用一条 SQL 同时做关系查询和向量搜索——比如"在 tech 分类中找与查询最相似的 10 篇文档"。

但 pgvector 不支持分布式（依赖 PostgreSQL 的单节点能力），当向量规模超过千万级时，查询性能会显著下降。它适合中小型项目、已有 PG 基础设施、需要关系+向量混合查询的场景。

"""pgvector 客户端：PostgreSQL 向量扩展"""
import psycopg2


class PgVectorClient:
    """
    pgvector 客户端：PostgreSQL 向量扩展。

    数据库端需要先安装扩展：
    CREATE EXTENSION vector;
    """

    def __init__(self, table_name="embeddings",
                 dsn="postgresql://localhost:5432/mydb"):
        self.table_name = table_name
        self.dsn = dsn
        self.conn = None

    def _init_client(self, dimension=768):
        """延迟初始化数据库连接并创建表"""
        self.conn = psycopg2.connect(self.dsn)
        with self.conn.cursor() as cur:
            cur.execute(f"""
                CREATE TABLE IF NOT EXISTS {self.table_name} (
                    id TEXT PRIMARY KEY,
                    embedding VECTOR({dimension}),
                    metadata JSONB
                )
            """)
            self.conn.commit()

    def insert(self, vectors, ids, metadata=None) -> int:
        """批量插入"""
        if self.conn is None:
            self._init_client()
        with self.conn.cursor() as cur:
            for i, vec in enumerate(vectors):
                vs = f"[{','.join(str(v) for v in vec)}]"
                m = metadata[i] if metadata and i < len(metadata) else '{}'
                cur.execute(
                    f"INSERT INTO {self.table_name} "
                    f"(id, embedding, metadata) VALUES (%s, %s, %s)",
                    (ids[i], vs, json.dumps(m)),
                )
            self.conn.commit()
        return len(ids)

    def search(self, query_vector, top_k=5):
        """
        向量搜索：使用余弦距离运算符 <=>
        pgvector 支持三种距离：
        - <=> 余弦距离
        - <#> 负内积
        - <-> 欧氏距离
        """
        if self.conn is None:
            return []
        vs = f"[{','.join(str(v) for v in query_vector)}]"
        with self.conn.cursor() as cur:
            cur.execute(
                f"SELECT id, metadata, embedding <=> %s AS dist "
                f"FROM {self.table_name} "
                f"ORDER BY dist LIMIT %s",
                (vs, top_k),
            )
            return [SearchResult(id=r[0], score=1.0/(1.0+r[2]),
                    metadata=r[1] or {}) for r in cur.fetchall()]

四、向量索引类型对比

向量数据库的性能很大程度上取决于所使用的索引类型。不同的索引在精度、速度和内存占用之间有不同的权衡。理解这些权衡是选型的关键一步。

索引类型原理精度速度适用规模
Flat（暴力搜索） 计算查询向量与所有向量的距离 100% 慢 < 100 万
IVF（倒排文件） 先聚类，只在相关簇中搜索 95-99% 快 百万到千万级
HNSW（小世界图） 多层图结构，逐层逼近最近邻 98-99.9% 很快 各种规模
PQ（乘积量化） 将向量分段量化，压缩存储空间 90-95% 最快 一亿以上
💡 索引选择建议

如果你不确定选哪个索引：先用 HNSW。它在精度和速度之间取得了最好的平衡，是大多数场景下的默认选择。只有当内存成为瓶颈（向量规模超过一亿）时，才考虑 PQ 量化来压缩存储。

五、存储成本估算

选择向量数据库时，存储成本是不可忽视的因素。每个 float32 占 4 字节，100 万向量 × 768 维 = 2.9 GB 纯向量数据。但不同数据库有不同的额外开销：

数据库存储大小额外开销原因
FAISS~2.9 GB纯向量存储，几乎无额外开销
Chroma~4.3 GB向量 + 元数据 + 内置文档存储
Qdrant~5.1 GBHNSW 图结构 + payload + WAL 日志
Milvus~5.7 GB索引 + 元数据 + 多副本
Pinecone~7.2 GB全托管，含索引和冗余备份
pgvector~5.7 GBPostgreSQL 页面开销 + TOAST 存储

实际开销通常是纯向量的 1-2.5 倍。规模越大，这个差距越明显。1000 万 × 1024 维时，FAISS 需要 38 GB，而 Pinecone 需要 95 GB——差了两倍多。所以在大规模场景下，选择轻量级的索引类型（Flat 或 IVF 而非 HNSW）可以显著降低存储成本。

六、选型决策指南

场景推荐方案原因
个人项目 / 快速原型 Chroma pip install 即用，零配置，适合开发阶段
单机生产 / 延迟敏感 FAISS 速度最快，内存效率最高

中小生产 / 需要过滤 Qdrant 性能好、过滤强、部署简单
大规模 / 企业级 Milvus 分布式、高可用、GPU 加速
不想运维 / 快速上线 Pinecone 全托管、自动扩展、零运维

七、常见误区

误区一：一开始就选最强大的

很多团队一上来就部署 Milvus 集群，结果发现 90% 的时间花在运维上，而 Chroma 或 FAISS 完全能满足当前需求。从简单开始，按需升级——先用 Chroma 验证方案，数据量增长后再迁移到 Qdrant 或 Milvus。

误区二：忽略向量维度对存储的影响

100 万 × 3072 维（OpenAI large）的存储成本是 100 万 × 768 维的 4 倍。在选择 Embedding 模型时，不仅要考虑精度，还要考虑向量维度对存储和检索性能的影响。有时候 768 维和 1024 维的效果差异很小，但存储成本差了三分之一。

误区三：向量数据库就是万能的

向量数据库擅长语义相似度搜索，但不擅长精确匹配、范围查询和事务处理。如果你的需求包含大量关系型查询，建议向量数据库 + 传统数据库组合使用，而不是试图用向量数据库替代一切。

八、小结

向量数据库的选型没有标准答案，关键是根据团队的技术能力、数据规模、运维资源、预算做出合理选择。对于大多数中文 RAG 项目，我们的建议是：开发阶段用 Chroma 快速验证，生产阶段用 Qdrant 或 Milvus 稳定运行。如果团队已有 PostgreSQL 基础，pgvector 也是一个值得考虑的低成本方案。
